# D353 — Dashboard WebRTC test calls don't appear in call logs

## Bug
Browser test calls from the dashboard "Talk to Your Agent" card and the auto-test after provision don't show up in:
1. Overview → Recent Calls section (shows "No calls yet")
2. Calls & Leads tab
3. `call_logs` table in Supabase (confirmed empty for this client)

The calls DO exist in Ultravox (confirmed via API — call 513c3578). But they're not inserted into `call_logs`.

## Root cause (likely)
The `agent-test` route (`/api/dashboard/agent-test/route.ts`) does insert to `call_logs` with `call_status='test'`. But the `browser-test-call` route and trial test-call route may not. Need to verify which route the dashboard orb uses.

Also: the auto-started test call after provision redirects to dashboard — which route does THAT use?

## Impact
- User does 3 test calls, sees "No calls yet" — thinks nothing is working
- Can't review transcript of test calls in the dashboard
- Can't validate that the agent is working before going live

## Fix
All test call routes should insert to `call_logs` with `call_status='test'`. Test calls should appear in Recent Calls with a "Test" badge.

## Files
- `src/app/api/dashboard/agent-test/route.ts` — check if logging
- `src/app/api/dashboard/browser-test-call/route.ts` — likely missing logging
- `src/app/api/trial/test-call/route.ts` — likely missing logging

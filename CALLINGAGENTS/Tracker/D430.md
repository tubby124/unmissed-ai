# D430 — No client-facing agent preview ("Test your agent" button)

**Status:** open
**Priority:** P2
**Filed:** 2026-04-06

## Problem
Clients have no way to hear what their agent says before a real caller hears it. First impression is on a live production call. Every other voice AI product has a test/preview mode.

## Root Cause
No UI feature built for this.

## Fix
Add a "Test Call" button to the dashboard that triggers a test call to the client's own phone number using the existing `/api/trial/test-call/route.ts` endpoint. Wire it up in the Settings page.

## Notes
The endpoint already exists. This is a UI-only addition. Low code complexity, high trust/onboarding value.

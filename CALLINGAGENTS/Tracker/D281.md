# D281 — CLOSE_PERSON (owner_name) editable post-onboarding

## Problem
After onboarding, `owner_name` is saved to DB but not synced to the prompt. The CLOSE_PERSON variable (first name of owner) appears ~20 times in the prompt — stale after name change.

## Solution (Phase 4)
1. Changed FIELD_REGISTRY: `owner_name` → `DB_PLUS_PROMPT` with `triggersPatch: 'owner_name'`
2. Added `'owner_name'` to `PATCH_TRIGGER_FIELDS` in `settings-patchers.ts`
3. Added `currentOwnerName` to `PatcherContext` + `fetchPatcherContext` select
4. Added owner_name patching block in `applyPromptPatches`:
   - Derives old first name from DB owner_name
   - Derives new first name from body.owner_name
   - Word-boundary replaces throughout prompt (reuses `patchAgentName()`)
   - Warning if old name not found in prompt

## Chain
body.owner_name → settings-patchers.ts → patchAgentName(oldFirst, newFirst) → system_prompt updated → computeNeedsSync → syncToUltravox

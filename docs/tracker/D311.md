# D311 — GBP Provenance Save Fix + Overview Knowledge Tile Merge

## Problem

Two bugs discovered during Muffin House Cafe end-to-end onboarding test:

1. **GBP provenance save fails silently** — `provision/trial/route.ts:437` used `void` (fire-and-forget) on the Supabase update for `gbp_place_id`, `gbp_rating`, `gbp_review_count`, `gbp_photo_url`. The `void` meant errors were swallowed. Result: GBP data collected during onboarding was never saved to the `clients` row.

2. **KnowledgeSourcesTile gates on `gbp_summary`** — `KnowledgeSourcesTile.tsx:166` checked `!!(localGbp?.summary)` but Google Places API returns `null` for `editorialSummary` on most businesses. GBP data (rating, reviews, photo) existed but was hidden because the summary was null.

3. **Overview showed no onboarding data** — `KnowledgeInlineTile` (used in Overview T4) only showed knowledge chunk counts, not the GBP profile data or business facts collected during onboarding.

## Fix

### Backend
- `provision/trial/route.ts:437` — Changed `void supa.from('clients').update(...)` to `await` with error logging
- Classification: **partial-failure bug** — DB write silently dropped

### Frontend
- `KnowledgeSourcesTile.tsx:166` — Changed gate from `!!(localGbp?.summary)` to `!!(localGbp?.placeId || localGbp?.rating)`
- `KnowledgeSourcesTile.tsx:286` — Added fallback text "Google Business Profile connected" when summary is null
- `KnowledgeInlineTile.tsx` — **Major merge**: Added GBP identity strip (photo + stars + reviews + website status badge) and business facts preview section. New props: `gbpData`, `businessFacts`, `faqCount`, `websiteUrl`, `websiteScrapeStatus`
- `UnifiedHomeSection.tsx` — Wired all new props from `data.gbpData` and `data.editableFields` to KnowledgeInlineTile

### Files Changed
- `src/app/api/provision/trial/route.ts` — `void` → `await` + error log
- `src/components/dashboard/home/KnowledgeSourcesTile.tsx` — GBP gate fix + null summary fallback
- `src/components/dashboard/home/KnowledgeInlineTile.tsx` — GBP strip + facts preview + new props
- `src/components/dashboard/home/UnifiedHomeSection.tsx` — Wire new props

## Verification
- Build passes clean (pre-existing test errors only)
- Muffin House Cafe manually patched with GBP data + business facts via SQL
- Future provisions will auto-save GBP data via the `await` fix

# D386 — Re-wire Niche Step into Onboarding Flow

## Problem
13 niche step files exist (`src/app/onboard/steps/niches/*.tsx`) but are ALL dead code. Phase 7 removed them from the step registry, leaving a 3-step flow: GBP → Plan → Launch. New clients get zero niche-specific configuration.

## What Needs to Be Done

### 1. Create `StepNiche` wrapper component
`src/app/onboard/steps/step-niche.tsx` — dynamically renders the correct niche file based on `data.niche`:
```tsx
const NICHE_COMPONENTS = {
  auto_glass: AutoGlassNiche,
  plumbing: PlumbingNiche,
  property_management: PropertyManagementNiche,
  // ... etc
}
export default function StepNiche({ data, onUpdate }: BaseStepProps) {
  const NicheComp = NICHE_COMPONENTS[data.niche] ?? GenericNiche
  // bridge: convert onUpdate(Partial<OnboardingData>) → onChange(key, value)
  const onChange = (key: string, value: unknown) =>
    onUpdate({ nicheAnswers: { ...data.nicheAnswers, [key]: value } })
  return <NicheComp data={data} onChange={onChange} />
}
```

### 2. Fix prop interface mismatch
All niche files use `onChange(key: string, value: string | string[] | boolean)`.
Step registry passes `onUpdate(updates: Partial<OnboardingData>)`.
The `StepNiche` wrapper bridges this — niche files don't need to change.

### 3. Add to step registry
`src/app/onboard/config/steps.ts` — insert between GBP and Plan:
```ts
{
  label: 'Setup',
  component: StepNiche,
  canAdvance: (d) => true, // optional step — always allow advance
}
```

### 4. Handle `niche='other'` case
When niche is undetected, show a generic "Tell us about your business" step with the 6 universal questions from NLM Q2. Don't skip the step — every client should go through it.

## Files to Create/Change
- `src/app/onboard/steps/step-niche.tsx` — NEW wrapper
- `src/app/onboard/config/steps.ts` — add StepNiche to STEP_DEFS

# D418 — PM infrastructure logistics chips (water shutoff, vendor access, showing instructions)

**Status:** 🔴 Open
**Priority:** MEDIUM
**Date:** 2026-04-05

## Problem
NLM research identified that PM agents sound generic because they don't know building-specific operational logistics. The current chip screen captures pet/parking/packages — but not the infrastructure details that make the agent feel custom-built for that property.

When a tenant calls about a burst pipe, the agent can't say:
> "The water shutoff is in the utility room on the east wall near the main entrance. I'll flag this P1 for your maintenance team."

Instead it just says "I'll have the manager call you back."

NLM Q5: "For property management: `{{infrastructure_logistics}}` — water shutoff locations, vendor access times, showing instructions per property" is the variable that creates the "this agent was built for our building" conviction.

## Solution
Add an **Infrastructure** section to `property-management.tsx` onboarding chip screen:

**Water shutoff location (optional freetext):**
```
Where is the main water shutoff? Tenants ask during emergencies.
e.g., "Utility room, east wall near main entrance"
```

**Vendor access (optional freetext):**
```
How should maintenance contractors access units?
e.g., "Lockbox code 4519 at leasing office, Mon-Sat 8am-5pm only"
```

**Showing instructions (optional freetext):**
```
How should prospects request a showing?
e.g., "Call leasing office during business hours — self-guided tours available with 24hr notice"
```

These are injected into `context_data` alongside the tenant roster as an INFRASTRUCTURE block, or as a named `injected_note` for PM clients.

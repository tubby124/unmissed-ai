# D298 — AI Compiler as Universal Knowledge Refinery

## Problem
The AI Compiler currently only processes website scrapes and uploaded documents. But there are 7 input sources that all produce unstructured text that needs to become agent knowledge. Each source has its own ad-hoc processing path. This creates inconsistent quality, no trust-tiering on some paths, and raw unformatted text reaching the agent.

## Vision
The AI Compiler becomes the SINGLE gateway for ALL knowledge ingestion:

```
ANY INPUT → AI Compiler → Trust-tier → GLM-4.6 format → Human review → knowledge_chunks
```

### Input Sources (all route through compiler)

| Source | Current Path | New Path |
|--------|-------------|----------|
| Website scrape | scrapeWebsite() → seedKnowledgeFromScrape() | scrape → **compiler** → review → seed |
| PDF/doc upload | embedChunks() direct | extract text → **compiler** → review → seed |
| GBP reviews | NOT IMPLEMENTED | GBP import → **compiler** → review → seed |
| Call learnings (gaps) | D252 gap detection → D257 suggestion (ad-hoc) | gap → **compiler** generates answer → review → seed |
| Manual text paste | reseedKnowledgeFromSettings() direct | paste → **compiler** → auto-approve (user typed it) |
| Onboarding form text | buildPromptFromIntake() direct | form → **compiler** → auto-approve for user input |
| Business description | D247 Haiku inference | description → **compiler** → review → seed |

### What the Compiler Does (for every source)

1. **Extract** — Haiku 4.5 pulls structured items (facts, FAQ, pricing, hours, policies)
2. **Classify** — Assigns kind (fact, faq_pair, pricing_or_offer, call_behavior_instruction)
3. **Trust-tier** — Based on source reliability:
   - User typed directly → `high` (auto-approve OK)
   - AI extracted from user's document → `medium` (needs confirm for HIGH_RISK_KINDS)
   - Auto-detected from call patterns → `low` (always needs approve)
4. **Format for GLM-4.6** — Rewrites into spoken voice format:
   - No bullet points, markdown, URLs
   - Short sentences (under 25 words)
   - Conversational phrasing
5. **Block dangerous items** — BLOCKED_KINDS never written:
   - `call_behavior_instruction` (prompt injection via document)
   - `unsupported_or_ambiguous` (unclear intent)
   - `conflict_flag` (contradicts existing approved knowledge)
6. **Present for review** — User sees extracted items, approves/edits/dismisses

### Governance Rules (already implemented, extend to all sources)
- HIGH_RISK_KINDS (`pricing_or_offer`, `hours_or_availability`, `location_or_service_area`, `operating_policy`) get `trust_tier='medium'` — UI shows verification checkbox
- BLOCKED_KINDS never written regardless of source
- `compile_run_id` FK tracks provenance (which compilation produced this chunk)
- User-approved items get `trust_tier='high'` and `status='approved'`

## Technical Notes
- Compiler already exists: `/api/dashboard/knowledge/compile` (extract) + `/compile/apply` (write)
- Need: generalize input to accept any text source (not just scrape/doc text)
- Need: add call-learning source type (gap question → compiler generates answer)
- Need: auto-approve path for user-typed content (skip review for `trust_tier='high'`)
- Need: GLM-4.6 formatting pass in compiler output (currently returns raw extracted text)

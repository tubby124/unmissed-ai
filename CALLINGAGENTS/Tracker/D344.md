# D344 — Niche promptfoo test registry

## Goal
Every production-ready niche gets a promptfoo test suite that validates:
1. Opening capability signal (Pattern 1)
2. Magic question per intent (Pattern 2)
3. Explicit outcome per intent (Pattern 3)
4. Urgency detection (Pattern 4)
5. NEVER rules enforced (Pattern 5)
6. Knowledge answers (if KB uploaded)
7. Info collection per intent (Pattern 7)
8. Universal: voice lock, security, 911, spam

## Pattern
For each new niche:
1. Create `clients/{test-slug}/config.json` + `domain-knowledge.md`
2. Write `tests/promptfoo/scripts/generate-{slug}-prompt.ts`
3. Write `tests/promptfoo/{slug}-test.yaml` following the framework in [[Niche-Test-Framework]]
4. Run, document findings, iterate

## Priority
Start with niches that have the most different "magic question":
1. plumbing (urgency = emergency dispatch)
2. dental (new vs existing patient)
3. hvac (system completely out vs not working right)

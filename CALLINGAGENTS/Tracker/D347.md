# D347 — CLOSE_PERSON fallback gets wrong value when no owner name

## Bug
`provision/trial/route.ts` line 91:
```ts
const ownerFirstName = (data.ownerName || data.businessName || '').split(' ')[0]?.trim()
```
When GBP auto-fills (no owner name collected), falls back to business name → "Red" from "Red Swan Pizza". Prompt then says "I'll make sure Red follows up" which sounds broken.

## Options
1. **Skip CLOSE_PERSON** if only business name available — let prompt use "the owner" default
2. **Use agent name** as fallback: "I'll make sure Sofia follows up" — sounds natural
3. **Add owner name field** to step 1 or step 3 — adds friction
4. **Infer from GBP** — Google Places doesn't expose owner name

## Recommendation
Option 2: fallback chain = `ownerName.split(' ')[0]` → `agentName` → skip. One-line fix.

# D420 — PM weekly digest: structured P1/P2/P3 + showing request summary

**Status:** 🔴 Open
**Priority:** MEDIUM
**Date:** 2026-04-05

## Problem
The weekly digest cron (`/api/cron/weekly-digest-client`) exists but sends generic call stats. For PM clients it should send a structured summary of what Jade handled that week — PM-specific metrics that a property manager actually cares about.

A good PM weekly digest in Telegram/email:
```
📋 Weekly Summary — Bowness Property Management
Week of Apr 7–13, 2026

🚨 P1 Emergencies: 2 (both transferred to emergency line)
⚠️ P2 Urgent: 5 (all logged for follow-up)
🔧 P3 Routine: 12 (logged, awaiting scheduling)
🏠 Showing Requests: 8 (4 prospects, avg: weekday evenings)
📞 Total calls handled: 31
✅ Missed calls caught by Jade: 31

Top issues this week: heating (4), plumbing (3), appliance (3)
```

vs the current generic version:
```
Your agent handled 31 calls this week. 8 HOT leads.
```

## Solution
For PM niche clients, parse the weekly call logs to extract:
- Count of [P1 URGENT], [P2 URGENT], [P3 ROUTINE] tags from AI summaries
- Count of [SHOWING REQUEST] tags + extract showing time preferences mentioned
- Issue category breakdown (from maintenance request categories if D405 is implemented, otherwise parse AI summaries)
- Calls handled = 100% catch rate (this is the retention hook — remind them of the value weekly)

## Dependencies
- D405 (maintenance_requests table) makes category breakdown accurate
- Without D405: parse AI summary text for P1/P2/P3 tags (weaker but workable)

- `src/app/api/cron/weekly-digest-client/route.ts` — add PM-specific digest logic
- `src/lib/transcript-analysis.ts` — may need PM tag extraction function

# D285 — Prompt Sandwich Framework Spec

## Problem
No formal specification for the section order and composition rules of voice agent system prompts. The 829-line `buildPromptFromIntake()` function and 18-step post-build injection pipeline are undocumented.

## Solution
Created `docs/architecture/prompt-sandwich-spec.md` defining:
- 19 named slots (discovered 1 extra: OBJECTION_HANDLING)
- Strict top-to-bottom order
- Per-slot: type (static/dynamic/conditional), source DB fields, char budget, current patcher, missing data behavior
- Full variable registry (30+ template variables mapped to slots)
- 18-step post-build injection pipeline documented
- Char budget analysis: 4.5K minimal → 6.5K typical → 9K maximal → 12K ceiling
- Migration strategy for Phases 2-6

## Key Findings
- CONVERSATION_FLOW slot is largest + most complex — should decompose into sub-slots in Phase 2
- KNOWLEDGE_BASE marker wraps too far (includes calendar/SMS/VIP if present)
- TRANSFER_ENABLED string cleanup is fragile (literal "true"/"false" string munging)
- Two variable fill passes needed (could be eliminated by slot functions resolving internally)
- OBJECTION_HANDLING is an undocumented 19th slot — not in original 18-slot plan

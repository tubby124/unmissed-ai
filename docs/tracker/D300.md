# D300 — Service Catalog Knowledge Reseed

## Problem
D260 patches the prompt FAQ line when services change, but doesn't reseed pgvector knowledge chunks. For pgvector clients, service changes are invisible to the queryKnowledge tool.

## Solution (Phase 5)
Extended `syncServiceCatalogToPrompt()` in `service-catalog-sync.ts`:
- After DB save, if `knowledge_backend='pgvector'`, calls `reseedKnowledgeFromSettings(clientId, businessFacts, extraQa)`
- Reseeds all `settings_edit` source chunks (facts + QA) — ensures service context is embedded
- Fire-and-forget: errors logged but don't fail the parent service operation

## Files Changed
- `src/lib/service-catalog-sync.ts` — added import of `reseedKnowledgeFromSettings`, added step 5 (knowledge reseed), expanded select to include `business_facts` and `extra_qa`

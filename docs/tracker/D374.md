# D374 — Calendar checkbox → booking mode guard in generate-agent-intelligence

**Status:** ✅ Done
**Date:** 2026-04-04

## What
Added `calendarEnabled?: boolean` to `AgentIntelligenceRequest` interface in `generate-agent-intelligence/route.ts`.

Added post-generation guard: when `calendarEnabled === false`, replaces `→ Outcome: book appointment` and `bookCalendar` references in the generated `TRIAGE_DEEP` with callback language:
> "I've got all those details logged. I'll have {{CLOSE_PERSON}} call you back to get you on the schedule — does that work?"

## Why
NLM Q4 (notebook 8d82f254) confirmed that agents without a connected calendar must never negotiate a time slot — it risks hallucinating availability or making false promises. The architectural goal shifts from booking to intake + handoff.

The guard runs at generation time so the stored `TRIAGE_DEEP` already has the correct routing before the prompt is assembled.

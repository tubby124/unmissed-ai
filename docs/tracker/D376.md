# D376 — Telegram Gated at Activation

## Summary
Amber warning on the final onboarding step if Telegram isn't connected, so operators don't go live blind.

## What Was Done (2026-04-04)
Added unconditional amber warning block in `step6-activate.tsx` after the notification method selector.

The warning is always shown (not conditional on notification method) because `telegram_chat_id` is always null pre-activation — the connection happens after the client is provisioned.

Message: "Telegram not connected — you won't receive lead notifications until you connect it. Connect it in Settings after going live."

This is NOT a hard block — operator can still launch. Amber warning only.

- `src/app/onboard/steps/step6-activate.tsx` — amber warning block after `<select>` for notificationMethod

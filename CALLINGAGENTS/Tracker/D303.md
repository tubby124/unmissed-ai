# D303 — Variable Edit API

## Problem
29 prompt variables are locked post-onboarding. No API path to edit them from the dashboard.

## Solution
`PATCH /api/dashboard/variables` — accepts `{ variableKey, value }`, routes to correct storage:
- If variable has a `dbField` (e.g. AGENT_NAME → `clients.agent_name`): writes to that column
- If variable has no dbField (niche-sourced): merges into `niche_custom_variables` JSONB
- Then calls `regenerateSlots(clientId, affectedSlots)` — only affected slots, not full recompose

Uses the D283a `prompt-variable-registry.ts` to map variable → DB field → affected slots.

## Key files
- `src/app/api/dashboard/variables/route.ts` — PATCH handler
- `src/lib/prompt-variable-registry.ts` — variable → storage mapping

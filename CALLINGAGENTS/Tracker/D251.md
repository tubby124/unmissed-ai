# D251 — Per-Section Prompt Editor UI

## Problem
When the agent says something wrong, the owner can't fix it themselves.
Everything routes through Hasan doing `/prompt-deploy` manually.
This is the bottleneck that prevents scale.

## Status: Backend Done
- `replacePromptSection()` in `lib/prompt-sections.ts` ✅
- `PATCH /api/dashboard/settings` accepts `section_id + content` ✅
- Missing: UI

## UI Needed
Render each prompt section as an expandable edit block in the settings Agent tab:
- OPENING
- TRIAGE
- CALL HANDLING MODE
- INFO COLLECTION
- AFTER HOURS
- ESCALATION
- EDGE CASES

Each block:
- Shows current section content (read from `clients.system_prompt` markers)
- Edit textarea
- Save button → PATCH with `section_id + content`
- Reset to default button

## Files
- `src/components/dashboard/settings/AgentTab.tsx` — layout shell, add sections here
- New `src/components/dashboard/settings/PromptSectionEditor.tsx` — the edit block component
- `src/lib/prompt-sections.ts` — backend already done

## Connection to D252
When D252 surfaces "missed pricing 3x" → owner clicks → opens TRIAGE section in this editor.
This is the self-serve fix loop that removes Hasan as bottleneck.

## Acceptance
- Owner can view and edit each prompt section from the dashboard
- Save triggers `replacePromptSection()` → agent resyncs via `needsAgentSync`
- Reset restores default from `niche-defaults.ts`

# D358 — Inline variable editing from Overview/Knowledge

## Problem
Users can't edit agent name, business name, services, or other prompt variables from the Overview or Knowledge pages. These are the "lego pieces" from Phase 6 (prompt slots + variables) but there's no frontend surface to edit them.

## Current State
- `PATCH /api/dashboard/variables` exists (D303 ✅)
- Variable registry with 39 variables exists (D283a ✅)
- But NO UI component exposes these for editing
- D283b (PromptVariablesCard) is still NOT STARTED

## Fix
Build the PromptVariablesCard on Settings → Agent tab showing:
- Agent name (editable)
- Business name (editable)
- Opening line (editable)
- Close person (editable)
- Services offered (chip editor)
- Urgency triggers (chip editor)
- TRIAGE_DEEP intents (read-only display)
- NEVER rules (read-only display)

Also add inline edit buttons on Overview KnowledgeInlineTile for key fields.

## Files
- `src/app/api/dashboard/variables/route.ts` — already built
- `src/lib/prompt-variable-registry.ts` — variable definitions
- NEW: `src/components/dashboard/settings/PromptVariablesCard.tsx`

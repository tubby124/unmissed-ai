# D360 — Raise prompt char limit to 25K

## Problem
The intelligence seed generates prompts ~18K chars. The `validatePrompt()` 12K limit only fires on settings PATCH (not provision). User confirms 25K is acceptable — prefer shorter but OK with longer when KB queries handle the rest.

## Fix
1. Update `validatePrompt()` in `src/lib/prompt-validator.ts` to allow 25K chars
2. Add warning at 15K (soft limit) — "Prompt is long, consider moving details to knowledge base"
3. Hard reject at 25K

## User's Words
"I think it's fine we can go up to 25K characters. I prefer to keep it shorter and tighter and allow it to query the knowledge base and get smarter from there."

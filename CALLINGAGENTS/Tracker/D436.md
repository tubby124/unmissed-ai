# D436 — PM: VIP Contacts / Tenant Caller ID Onboarding Guide

## Status
OPEN — documentation gap, no code bug

## Problem
The `client_contacts` table supports a `is_vip` flag. `buildAgentContext()` already pulls VIP contacts and injects them as caller context at call time — meaning if a tenant calls from their registered number, the agent already knows their name.

PM clients are never told about this feature during onboarding or in the dashboard. They're aware of the tenant roster textarea (which injects into TRIAGE_DEEP as a text block), but not the structured contacts system that enables live caller ID recognition.

The two systems are distinct:
- **Tenant roster** (`nicheAnswers.tenantRoster`) → pasted into TRIAGE_DEEP, static text reference
- **Client contacts** (`client_contacts` table, `is_vip=true`) → injected at call time via `buildAgentContext()`, enables real-time caller recognition by phone number

## Impact
PM clients with 10+ tenants who fill in the tenant roster textarea still won't get live caller ID unless they also add tenants to the contacts table. This is a missed product value that sounds like magic ("your agent knows it's you, Unit 4B") but requires setup we're not explaining.

# D359 — Smart KB→Prompt promotion

## Problem
If the agent queries the knowledge base 3+ times for the same topic, it's wasting latency on every call. The system should detect this pattern and suggest promoting that knowledge into the system prompt (where it's instantly available without a tool call).

## Design
1. In `POST /api/knowledge/{slug}/query` — log each query text to `knowledge_query_log` (already done)
2. Add a cron or post-call check: group `knowledge_query_log` by normalized query text, count occurrences
3. When count >= 3 for same topic cluster: create a suggestion in a new `knowledge_suggestions` table or surface it on the Unanswered Questions tile
4. Show on dashboard: "Your agent searches for 'delivery area' on every call. Add this to your agent's core knowledge?"
5. On accept: add as FAQ pair to `extra_qa` or inject into prompt via variable

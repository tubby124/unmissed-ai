# D413 — Capture emergency tech phone + after_hours_behavior during PM onboarding

**Status:** 🔴 Open
**Priority:** HIGH
**Date:** 2026-04-05

## Problem
The two fields that activate P1 emergency dispatch (`after_hours_emergency_phone` and `after_hours_behavior: route_emergency`) are only accessible post-onboarding in settings. A PM who signs up doesn't know to set these — they complete onboarding and Jade works, but P1 calls still can't transfer.

The onboarding chip flow asks about property types, unit count, pet policy, etc. — but never asks for the emergency tech contact.

## Solution
Add two new fields to the PM onboarding chip screen (`property-management.tsx`):

**Field 1 — Emergency tech phone:**
```tsx
<Label>Emergency tech number (for live transfer on P1 calls)</Label>
<p>This is who Jade calls at 2am when a tenant reports flooding, no heat, or a break-in.</p>
<input type="tel" placeholder="(416) 555-1234" onChange={e => onChange('emergencyTechPhone', e.target.value)} />
```

**Field 2 — After-hours behavior:**
```tsx
<ChipSelector
  options={[
    { value: 'route_emergency', label: 'Transfer P1 emergencies immediately (24/7)' },
    { value: 'take_message', label: 'Take messages only — PM reviews next business day' },
  ]}
  onChange={val => onChange('afterHoursBehavior', val)}
/>
```

On provision:
- `emergencyTechPhone` → `after_hours_emergency_phone` in settings
- `afterHoursBehavior` → `after_hours_behavior` in settings

This way P1 transfer is configured during onboarding and works from day 1 of the trial.

## Files
- `src/app/onboard/steps/niches/property-management.tsx` — add fields
- `src/lib/intake-transform.ts` — map `emergencyTechPhone` and `afterHoursBehavior` to settings fields

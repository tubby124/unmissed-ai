# D260 — Service Catalog → Agent Disconnect

## Problem
Client edits services in `client_services` table (price=$25, duration=30min for toilet repair) but these changes never flow to the live agent's prompt. The service catalog is only used during `buildPromptFromIntake()` at onboarding time. Post-onboarding edits to services are dead data.

## Root Cause
`formatServiceCatalog()` in `service-catalog.ts` is only called by `buildPromptFromIntake()`. There's no `patchServicesOffered()` → settings PATCH → Ultravox sync chain for service catalog changes.

## Fix Plan
- When `client_services` are edited, rebuild `SERVICES_OFFERED` from active service rows
- Patch this into the prompt via `patchServicesOffered()` (already exists)
- Wire the service catalog save to trigger `needsAgentSync`
- Or: move service catalog to `{{contextData}}` injection (per-call, no prompt patch needed)

## Status
Open — high impact. Every service catalog edit is currently dead.

# D313 — Telegram Bot Link — Click Connected Opens Bot Chat

## Problem

The Telegram "Connected" pill in QuickConfigStrip was not clickable in a useful way — it opened the notifications sheet regardless of connection state. Users who already connected Telegram should be taken directly to their bot chat (same chat where they receive call alerts).

## Fix

- When `telegramConnected = true` AND `telegramBotUrl` exists: clicking opens the bot URL in a new tab (`window.open(url, '_blank', 'noopener')`)
- When `telegramConnected = false`: clicking still opens the notifications sheet for setup

### Data Flow
```
/api/dashboard/home → builds telegramBotUrl from env TELEGRAM_BOT_USERNAME
  → HomeData.onboarding.telegramBotUrl
    → UnifiedHomeSection → QuickConfigStrip telegramBotUrl prop
      → window.open() on pill click
```

Bot URL format: `https://t.me/{TELEGRAM_BOT_USERNAME || 'hassitant_1bot'}`

All clients use the same bot (`hassitant_1bot`). Each client's Telegram user has a private 1:1 chat thread with the bot, identified by `telegram_chat_id` on their `clients` row. Opening the bot URL takes the user to their existing chat.

### Files Changed
- `src/app/api/dashboard/home/route.ts` — Added `telegramBotUrl` field to onboarding response
- `src/components/dashboard/ClientHome.tsx` — Added `telegramBotUrl` to HomeData type
- `src/components/dashboard/home/UnifiedHomeSection.tsx` — Pass `telegramBotUrl` to QuickConfigStrip
- `src/components/dashboard/home/QuickConfigStrip.tsx` — New `telegramBotUrl` prop, conditional click handler

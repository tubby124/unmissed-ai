# D290 — "What Your Agent Knows" Surface

## Problem
No single place shows users what their agent actually knows. Facts are scattered across:
- `business_facts` (settings card)
- `extra_qa` (FAQ card)
- `knowledge_chunks` (invisible pgvector store)
- `client_services` (service catalog card)
- `context_data` (reference data card)
- Hardcoded niche FAQ defaults (invisible, in code)

The user has no idea which facts the agent will use when answering a call.

## Vision

### "What Your Agent Knows" Section (Overview or Knowledge page)
```
📚 What Alex knows about Prairie Plumbing

FACTS (6 items)
• Open Mon-Fri 8am-5pm, Sat by appointment
• Emergency service during business hours only
• Licensed and insured
• Service area: Saskatoon and surrounding area
• Free estimates on all jobs
• [Edit facts →]

FAQ (12 answers ready)
• "Do you handle emergencies?" → "yeah, during business hours..."
• "How much does it cost?" → "depends on the job, the boss'll..."
• "Do you come to me?" → "we come to you, all our work is..."
• [Show all 12] [Edit FAQ →]

SERVICES (5 active)
• Drain Cleaning • Water Heater • Burst Pipe • Sewer Backup • Bathroom Reno
• [Edit services →]

KNOWLEDGE BASE (from website + uploads)
• 23 knowledge chunks from website scrape (approved)
• 0 uploaded documents
• [View knowledge →] [Upload document →]

MISSING (gaps detected from recent calls)
⚠️ 3 callers asked about "water softener" — no answer available
⚠️ 2 callers asked about "financing" — no answer available
[Add answers →]
```

### How It Works
- Reads from: `business_facts`, `extra_qa`, `client_services`, `knowledge_chunks` (count + sources), `knowledge_query_log` (gaps)
- Groups by category
- Shows count + preview for each
- Links to the editing surface for each category
- Shows knowledge gaps from recent calls (D252 already detects these)

## Implementation
- Component: `AgentKnowledgeSummary.tsx` (new, or extend existing)
- API: existing endpoints sufficient — `/api/dashboard/settings` GET, `/api/dashboard/knowledge/gaps` GET
- Place on: Overview page (prominent) + Knowledge page (detailed)
- Later (Phase 6): becomes part of Agent Brain page

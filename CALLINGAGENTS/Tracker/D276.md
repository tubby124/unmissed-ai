# D276 — Calendar/Booking Auto-Updates Call Flow

## Problem
When a user enables Google Calendar booking, the system prompt's call flow doesn't automatically update to lead with scheduling. The booking tool gets registered but the TRIAGE section still does generic qualifying instead of guiding toward appointment booking.

## Fix Plan
- When `booking_enabled` toggles ON → auto-patch CALL HANDLING MODE section to appointment_booking flow
- Include calendar availability check in the triage script
- The call flow should reflect: "Let me check what times work" instead of "I'll have someone call you back"
- Already partially done via `patchCalendarBlock()` — but the TRIAGE section doesn't adapt

## Status
Open — depends on D280 (UI-driven prompt composition) for the full vision.

# D265 — Remove PRODUCT KNOWLEDGE BASE from prompt-builder

## Problem
`buildPromptFromIntake()` generates a hardcoded PRODUCT KNOWLEDGE BASE section that duplicates information already in `knowledge_chunks` (pgvector). This wastes prompt tokens and creates a stale copy of knowledge that doesn't update when the knowledge base changes.

## Fix Plan
- Remove the static PRODUCT KNOWLEDGE BASE section from `template-body.ts`
- Knowledge should come from pgvector (`queryKnowledge` tool) at call time, not baked into prompt
- Keep `{{businessFacts}}` injection for lightweight summary, but remove the heavy block

## Status
Open — part of the prompt compression / minimal base prompt initiative (D268).

# D409 — Progressive disclosure for settings dashboard (Essential vs Advanced)

**Status:** 🔴 Open
**Priority:** MEDIUM
**Date:** 2026-04-05

## Problem
The settings dashboard renders 20+ cards in a flat list. Non-technical users (PMs, salon owners, contractors) open it and don't know where to start. The most important cards (agent config, hours, forwarding number) compete visually with advanced cards (prompt editor, god mode, prompt variables, webhook config).

NLM research finding (D280): "Progressive disclosure solves the 'too many settings' problem. New users see 3-4 essential filling cards. Power users unlock the full view." Current flat list contradicts this.

## Solution
Group cards into two sections in `AgentTab.tsx`:

**Essential (always visible):**
- Agent config (name, voice)
- Hours
- Services / knowledge
- Call routing / forwarding number
- Test call
- Niche-specific card (PM setup checklist, etc.)

**Advanced (collapsed by default, toggle to reveal):**
- Prompt editor
- Prompt variables
- Voice style presets
- Booking / calendar
- Webhooks
- God mode
- Learning loop
- Prompt suggestions

Show "Advanced Settings" toggle button at the bottom of the Essential section. When expanded, show all Advanced cards. State persists in localStorage.

## Files
- `src/components/dashboard/settings/AgentTab.tsx` — grouping + toggle logic
- No new components needed — just reorganize what's rendered

## Notes
Don't change any functionality — only grouping. Power users still get everything.
This is a perception fix, not a feature fix.

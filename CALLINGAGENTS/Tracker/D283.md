# D283 — All Prompt Variables Visible + Editable on Dashboard

## Problem
Users can't see what template variables are in their agent's prompt. They don't know what `{{TRIAGE_DEEP}}` contains, what `{{SERVICES_OFFERED}}` looks like, or what `{{PERSONALITY_LINE}}` says. They have no visibility into what drives their agent's behavior.

## Sub-tasks (Phase 5 split)

### D283a — Prompt Variable Registry ✅ DONE (2026-03-31)
Created `src/lib/prompt-variable-registry.ts`:
- 39 variables registered with full metadata
- Fields: key, label, slotId, dbField, intakeField, editable, editPath, category, nicheSourced, description
- Lookup helpers: getVariable, getVariablesForSlot, getVariablesByCategory, getEditableVariables, getSlotsAffectedByDbField
- Categories: identity (7), voice (4), flow (13), knowledge (6), capability (4), safety (3)

### D283b — Variable Dashboard Surface (read-only) — NOT STARTED
PromptVariablesCard.tsx — read-only display grouped by category, links to existing settings cards.

### D283c — Section-Level Regeneration ✅ DONE (2026-03-31)
Created `src/lib/slot-regenerator.ts`:
- `regenerateSlot(clientId, slotId)` — regenerates one slot section from current DB state
- `regenerateSlots(clientId, slotIds[])` — batch regeneration with single DB read
- Converts client DB row → synthetic intake → buildSlotContext → slot function → replacePromptSection
- Validates, saves, tracks prompt version, syncs to Ultravox
- Replaces the need for individual regex patchers for most variable changes

## Status
D283a + D283c DONE. D283b deferred to Phase 5 UI wave (D286/D288/D290).

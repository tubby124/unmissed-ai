# D235 — Knowledge Reseed Gate Removal

## Problem
`reseedKnowledgeFromSettings()` has a guard that skips reseeding when knowledge_chunks
already exist (original intent: avoid duplication on first seed).

But this means when a client updates business_facts or extra_qa in settings,
the changes DON'T propagate to pgvector until next full wipe.

## Solution
3-line change in `lib/embeddings.ts`:
Always delete 'settings_edit' source chunks before re-embedding, not just on first seed.

## File
`agent-app/src/lib/embeddings.ts` — reseedKnowledgeFromSettings()

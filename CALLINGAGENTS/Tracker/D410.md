# D410 — Agent Brain Gap Detection for PM

## Problem
PM agents frequently say "I don't have that information" for questions that SHOULD be answered (pet policy, parking, package handling) but weren't filled in during onboarding. No automated way to detect which context blocks are missing vs. intentionally blank.

## Fix
- Add a PM-specific completeness check in the setup checklist (D407 already has the checklist shell)
- Flag empty: pet policy, parking policy, package handling, maintenance contacts, emergency phone
- Show a "% complete" or "N of 5 fields filled" indicator in the PmSetupChecklist
- Optionally: surface in the drift-check cron to flag PM clients with empty critical fields

## Files
- `src/components/dashboard/settings/PmSetupChecklist.tsx` — add field completeness scoring
- `src/app/api/cron/drift-check/route.ts` — optional: add PM completeness check

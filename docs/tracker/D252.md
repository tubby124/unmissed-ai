# D252 — Knowledge Gap → One-Click Fix CTA

**Status:** ✅ DONE (2026-03-31)

## What Was Built

Three changes:

### 1. Home API (`src/app/api/dashboard/home/route.ts`)
- Added `gapTextsRes` query: fetches up to 50 unanswered query_text values from `knowledge_query_log` (last 30 days, `result_count=0`, `resolved_at IS NULL`)
- Aggregates by frequency (normalized lowercase), returns top 5 as `topGaps: { query_text, count }[]`
- Added to `insights` response object alongside existing `openGaps` count

### 2. `ClientHome.tsx` type
- Added `topGaps?: { query_text: string; count: number }[]` to `insights` field in `HomeData` interface

### 3. `UnifiedHomeSection.tsx` — KnowledgeGapCTA inline component
- Shows after `AgentReadinessRow` (D249)
- Only renders when `topGaps` has entries
- Displays top 2 questions with frequency badge (e.g. "3×") and "Answer →" link to `/dashboard/knowledge?q={encoded}`
- Shows "+N more" count with link to `/dashboard/knowledge`
- Header: "Missed questions" with "Teach agent →" CTA

## Before
- `openGaps` was just a count: "3 caller questions your agent couldn't answer"
- No way to know WHICH questions without going to another page

## After
- Shows the actual question text: `"what is your pricing?" (3×)` → `Answer →`
- Specific fix link pre-fills the knowledge page with the query

- `src/app/api/dashboard/home/route.ts` (gapTextsRes query + topGaps aggregation)
- `src/components/dashboard/ClientHome.tsx` (type update)
- `src/components/dashboard/home/UnifiedHomeSection.tsx` (KnowledgeGapCTA block)

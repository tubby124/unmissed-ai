Three Technical Pillars

1. Inventory Lookup — Ultravox RAG Corpus CONFIRMED

How it works: Eagle Head exports their inventory (CSV, text, or database dump) on a schedule. We upload it to an Ultravox corpus. The AI automatically searches it when a caller asks about a part.

SpecDetail
APIPOST /api/corpora → create corpus → add sources → documents auto-chunked/vectorized
Agent integrationBuilt-in queryCorpus tool — no custom HTTP tool needed
FormatsPDF, Word, EPUB, PowerPoint, Markdown, plain text, HTML
Limits20 sources/corpus, 200 docs/source, 10MB/doc
RefreshPATCH source to recrawl. Corpus stays queryable during refresh (stale until done).
Query results1-20 ranked chunks with content, score, citations
LatencyNot published — designed for real-time, presumably sub-second (server-side)

RAG Advantages

  • No API from Eagle Head's software needed
  • Just needs a data export (CSV, text file, anything)
  • Ultravox handles search natively — zero custom code
  • Works even if their POS is a 20-year-old desktop app

RAG Limitations

  • Data freshness depends on export frequency (hourly? nightly?)
  • Parts sold since last export may show as available
  • Solved by the staff confirmation gate (see Phase 1 flow below)

If supplier exposes an API: We swap RAG for an HTTP tool (proven transferCall pattern). Real-time inventory, zero staleness. RAG is the fallback — API is the upgrade path.

2. Payment — SMS Payment Link PCI COMPLIANT

How it works: Instead of the AI taking a credit card number over the phone (PCI nightmare), we send a Stripe Checkout link via SMS. Customer taps, pays on their phone. No card data ever touches our system.

1AI captures order details on the call
2Staff confirms part is still on the shelf (SMS/WhatsApp: tap YES/NO)
3YES → Stripe Checkout link auto-sends to customer via SMS
4Customer taps link, pays on their phone (30 seconds)
5Stripe webhook confirms payment → SMS to customer: "Payment received, your order is confirmed"
6Staff notified to pull the part

PCI scope: SAQ-A (lowest). No card numbers in voice transcripts, no card data on our servers. Stripe handles all payment processing. This is the industry-standard approach for voice AI + payments.

3. Order Entry — Post-Call RPA VIABLE BUT COMPLEX

How it works: After the call ends and payment is confirmed, a background process enters the order into Eagle Head's POS system. The customer never waits — it happens in the background.

ApproachSpeedReliabilityBest For
Claude Computer Use 2-10s per step, 30-120s total 70-90% Legacy desktop POS with unpredictable UI
Playwright/Puppeteer <1s per step, 5-15s total High (but brittle on UI changes) Web-based POS with stable interface

Mid-call RPA is NOT viable. A full order entry takes 30-120 seconds. Our mid-call latency budget is 1-2 seconds. The caller would be sitting in dead air. RPA only works as a post-call background process.

Recommended Phased Approach

1
Smart Agent + RAG Inventory + Staff Confirmation Gate NOT A GLORIFIED IVR

On the Call:

1AI answers: "Hey, you've reached Eagle Head Auto, how can I help?"
2Customer: "I need a brake caliper for a '99 F-150"
3AI searches inventory via queryCorpus (sub-second, server-side)
4AI: "We've got that listed at $45 — I've got all your details. Our team's gonna confirm it's still on the shelf and text you a payment link within the next few minutes."
5Call ends. Customer got a real answer, not "we'll call you back."

After the Call (Confirmation Gate):

6Staff gets instant notification: "New order: Brake caliper, '99 F-150, $45. Customer: John Smith (403-555-1234). Still available? Reply YES or NO"
7YES → Stripe payment link auto-sends to customer via SMS
8NO → Customer gets SMS: "Sorry, that part just sold — we'll let you know if another comes in"
9Customer pays via Stripe link → confirmation SMS → staff pulls part

Why the confirmation gate matters

  • Prevents ghost-part payments — RAG data can be stale (exported hourly/nightly, not real-time)
  • Without gate: customer pays for a part sold since last export → refund headache
  • With gate: staff verifies in 10 seconds (one tap) before payment link sends → zero ghost payments
  • As inventory sync improves (real-time API in Phase 2), gate becomes optional for high-confidence matches

Why this isn't a "glorified IVR": The customer gets a real inventory answer on the call (not "we'll call you back"). Payment link arrives in minutes. Staff's only action = one tap to confirm + pull the physical part. Nobody calls anybody back.

2
Supplier API Integration IF AVAILABLE
  • Replace RAG corpus with HTTP tool hitting supplier's API
  • Real-time inventory — zero staleness — confirmation gate becomes optional
  • Same call flow, just swapping the data source
  • Proven pattern: same architecture as our transferCall tool
  • Eagle Head is willing to ask their supplier. If they get API access, this is the upgrade path.
3
Post-Call Order Entry (RPA) BACKGROUND PROCESS
  • After payment confirmed: background RPA enters order into POS (30-120 seconds)
  • Claude Computer Use (legacy desktop POS) or Playwright (web-based POS)
  • Customer gets SMS when order is entered in the system
  • Staff reviews in POS — nothing ships without human checking
  • No caller waiting — all happens after the call ends
4
Full Automation (Barry's Vision) PHASES 1-3 COMBINED
  • Answer → check stock → quote → staff confirm → payment → order entry → pull part
  • Only human actions: (1) tap YES/NO to confirm availability, (2) pull the physical part
  • This IS "carry to completion" — the AI handles the entire customer-facing workflow
  • Even Amazon has humans in the warehouse. The physical pull always requires a person.

Meeting Talking Points (revised for Barry)

SAY

  • "We train the AI on your actual inventory — it checks stock live on the call and quotes the price"
  • "After the call, your team gets a one-tap confirmation — hit yes, customer gets a payment link instantly"
  • "No credit cards over the phone — customer pays on their own device, fully secure"
  • "Once they pay, we can enter the order into your system automatically in the background"
  • "Your staff just pulls the part. The phone work is done."
  • "Nobody has built this for salvage yards — you'd be the first"

DON'T SAY

  • "The AI controls your computer screen live on the call" — too slow mid-call (30-120s). Say "background process" instead.
  • "We'll search car-part.com automatically" — no API access
  • "The AI takes the credit card over the phone" — PCI risk. Say "payment link via text" instead.
  • "Real-time inventory" — say "trained on your latest inventory data" (honest about refresh cycle)
  • "100% automated ordering" — say "one tap to confirm, then automated from there"

Questions to Ask

  1. "What inventory software do you run?" (Checkmate? Pinnacle? Custom?)
  2. "Can you export your inventory as a file?" (CSV, text, any format — this is what feeds the RAG)
  3. "How often does your inventory change?" (determines refresh frequency)
  4. "Will you ask your software vendor about API access?" (Phase 2 unlock)
  5. "Is your POS web-based or a desktop app?" (determines RPA approach)
  6. "What did you mean by 'YouPick'?" (not a real platform — clarify)
  7. "Where should staff confirmations go?" (SMS, WhatsApp, email?)
  8. "Are prices fixed per part, or does someone approve pricing?"

Bottom Line

Phase 1 answers Barry's core objection. This is NOT a glorified IVR. The customer gets a real inventory answer on the call, pays via text within minutes, and staff's only job is one tap + pull the part. No callbacks. No phone tag.

Phase 2 removes the staleness risk. If the supplier exposes an API, inventory queries become real-time and the confirmation gate becomes optional. Eagle Head is willing to ask — this is worth pursuing.

Phase 3 closes the loop on order entry. Post-call RPA enters the order into POS automatically. 70-90% reliability is workable with a human review step. This is the "carry to completion" Barry asked for.

External yard search (car-part.com) is the one thing we genuinely can't do. No API exists. eBay Motors is the only option and covers a fraction of inventory. Be upfront about this.

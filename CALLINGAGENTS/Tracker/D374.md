# D374 — Calendar connect → auto-upgrade prompt to booking mode

## Problem

If a client connects Google Calendar post-onboarding, the prompt doesn't change. They still have a message-taking agent. To get booking capability they'd need a manual redeployment.

This breaks the expected UX: "I connected my calendar, why isn't it booking?"

## Fix

When a client's calendar is connected (or `calendar_enabled` flips to true in settings):
1. Trigger `recomposePrompt()` with `agent_mode = 'appointment_booking'` injected
2. Auto-deploy the new prompt to Ultravox
3. Show a confirmation banner: "Your agent can now book appointments directly on calls"

If calendar is later disconnected → revert to previous mode (store pre-booking mode in DB).

## Files
- `src/app/api/dashboard/settings/route.ts` — detect calendar connect, trigger recompose
- `src/lib/prompt-slots.ts` — ensure appointment_booking mode fires correctly
- `src/components/dashboard/settings/BookingCard.tsx` — show banner on connect

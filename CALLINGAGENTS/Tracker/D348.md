# D348 — .md file upload rejected by Knowledge document uploader

## Bug
The "Upload Document" modal on the Knowledge page says "Add PDFs or Word docs" and rejects .md (markdown) files. This is a problem because:
1. Many businesses have menus/info in markdown or plain text
2. Our own test data (`clients/sals-ny-pizza/domain-knowledge.md`) is markdown
3. Text/markdown is actually the easiest format for Haiku to parse

## Observed
User tried to upload a .md menu file → rejected. Had to save as .txt instead.

## Fix
Expand accepted file types in the upload component to include:
- `.md` (markdown)
- `.txt` (plain text)
- `.csv` (structured data)
- `.pdf` (already accepted)
- `.docx` (already accepted)

## Files
- `src/components/dashboard/knowledge/FileUploadPanel.tsx` or equivalent — file type accept list
- May also need backend route update if server-side validation rejects the MIME type

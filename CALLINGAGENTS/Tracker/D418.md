# D418 — Infrastructure / Logistics Chips in PM Onboarding

## Problem
PM onboarding (D413) added emergency phone and after-hours behavior. Additional logistics fields that affect Jade's behavior are not yet collected: laundry location, parking assignment process, garbage collection day, mailroom access.

## Fix
Add a second "Logistics" chip section to the PM onboarding screen, after the emergency fields. Fields:
- Laundry: in-unit / shared / none (ChipSelector)
- Parking: assigned / first-come / street (ChipSelector)
- Garbage day: text field (e.g., "Tuesday mornings")
- Mailroom access: key / lobby / package locker (ChipSelector)

Map each to `niche_*` keys in intake-transform.ts following the D413 pattern.

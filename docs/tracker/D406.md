# D406 — Maintenance requests dashboard view

**Status:** 🔴 Open
**Priority:** HIGH
**Date:** 2026-04-05
**Depends on:** D405 (maintenance_requests table + API)

## Problem
No UI for a PM client to see what maintenance requests Jade collected. Even after D405 wires up the DB, there's no way to view, filter, or act on requests from the dashboard.

## Solution
Add a **Requests** tab or card visible only to PM niche clients.

### Option A: New route `/dashboard/requests`
- Table view: unit, tenant name, issue category, urgency, date, status
- Filter by status (new / in_progress / completed)
- Click row to expand: full description, access window, entry permission, phone
- Status dropdown per row: mark as scheduled / in_progress / completed

### Option B: `MaintenanceRequestsCard` in settings
- Simpler: a card on the Agent tab (below other PM cards) showing recent requests
- "View all" link to the full table

Recommendation: Option A (dedicated route) — PMs will check this daily, it needs its own page.

### Badge/notification
- Show unread count badge on the Requests nav item when new requests arrive
- Tie into existing Telegram notification (already fires [P2] and [P3] tags)

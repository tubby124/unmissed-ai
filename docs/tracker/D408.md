# D408 — VIP contacts onboarding for PM (caller ID → tenant lookup)

**Status:** 🔴 Open
**Priority:** MEDIUM
**Date:** 2026-04-05

## Problem
The `client_contacts` table with `is_vip` flag already exists. `buildAgentContext()` already pulls the VIP roster and injects it as caller context at call time — meaning if a tenant calls from their registered number, the agent already knows their name before they say it.

PM clients are never told this feature exists. It's one of the most compelling PM features (agent greets: "Hi Sarah, I see you're calling from unit 4A — what can I help you with?") but it's completely invisible.

## Solution
Add a **VIP Contacts** onboarding step or settings card for PM niche specifically:
- "Add your tenants as contacts — Jade will recognize their number and greet them by name"
- Simple form: Name + Phone + Unit number
- On save → insert to `client_contacts` with `is_vip=true`
- Show current count: "12 tenants recognized by caller ID"

The `VIPContactsCard` in settings already exists (`src/components/dashboard/settings/VIPContactsCard.tsx`). May only need to:
1. Surface it more prominently for PM niche
2. Rename label to "Tenant Contacts" (not "VIP Contacts")
3. Add unit number field to the contact form
4. Add it as a step in D407's PM Setup Checklist

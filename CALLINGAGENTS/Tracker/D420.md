# D420 — PM Weekly Digest

## Problem
PM clients get real-time Telegram notifications per call (D415) but have no weekly summary: total calls, maintenance requests submitted, urgency breakdown, unresolved requests.

## Fix
- Add a weekly cron (Sunday 7am) that queries `maintenance_requests` + `call_logs` for each PM client
- Build a PM-specific digest message: N calls this week, N maintenance requests (X urgent, Y routine), N unresolved
- Send via existing Telegram bot
- Opt-out flag in client settings (default: on)

## Files
- `src/app/api/cron/weekly-digest-client/route.ts` — add PM branch
- `src/lib/completed-notifications.ts` — add `buildPmWeeklyDigest()` function
- `src/components/dashboard/settings/PmConfigCard.tsx` — add weekly digest toggle

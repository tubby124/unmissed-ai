# D350 — Knowledge source drawers don't expand on click

## Bug
Knowledge source cards had `>` chevron but clicking did nothing. User uploads menu → "12 chunks" → clicks → nothing → can't verify what was extracted.

## Fix (2026-04-01)
Wired source card clicks to open a Sheet/drawer with ChunkBrowser pre-filtered to that source.

**3 files changed:**

1. `KnowledgeSourceRegistry.tsx` — added `onSourceClick` prop. Active source cards (chunks > 0) render as `<button>` calling `onSourceClick`. Inactive cards remain as `<Link>` to add-knowledge page.

2. `ChunkBrowser.tsx` — added `initialSourceFilter` prop to pre-set the source dropdown filter.

3. `KnowledgePageView.tsx` — added `sourceDrawerFilter` + `sourceDrawerLabel` state. `openSourceDrawer()` handler maps source cards to DB source keys:
   - Single-key sources (Website, Facts & Q&A, AI Compiler, Documents, Google Profile) → pre-filtered drawer
   - Multi-key sources (Text Imports: bulk_import, dashboard_manual, manual, manual_text) → unfiltered drawer

No API changes needed — existing `GET /api/dashboard/knowledge/chunks?source=...` already supports filtering.

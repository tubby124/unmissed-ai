# D240 — Niche Intent Audit: Are Agents Answering or Routing?

## Problem
All active clients have info-bot prompts. They answer questions but have no intent classification or purpose-driven routing. A caller wanting to book doesn't get routed to booking — they get general answers until the agent remembers it has a booking tool.

## Action
Audit all 4 active client prompts for:
- Is there a PRIMARY GOAL section?
- Does the TRIAGE section route by intent?
- Does it classify: booking vs info vs transfer vs complaint vs junk?

## Files
- `src/lib/prompt-builder.ts` — TRIAGE section template
- `clients/*/SYSTEM_PROMPT.txt` — live prompts
- `CALLINGAGENTS/Product/Intent Classification.md`

## Acceptance
- Each client prompt has explicit intent routing in TRIAGE
- PRIMARY GOAL maps intent → action (not just "help callers")

# D352 — Knowledge query lag during calls

## Problem
When the agent calls `queryKnowledge`, there's a noticeable 1-2 second pause. The agent goes silent while waiting for the tool response. On the Red Swan test call, this happened 3 times (drinks, slices, appetizers).

## Possible causes
1. **pgvector query latency** — hybrid search (embedding + text) takes time
2. **Network round-trip** — Ultravox → Railway → Supabase → back
3. **No filler speech** — agent waits silently instead of saying "let me check..."

## Options
1. **Filler speech** — Ultravox `defaultReaction` on the queryKnowledge tool could say "let me check on that..." while the tool runs. Already configured in the tool definition but may not be working.
2. **Cache hot queries** — pre-compute answers for top 10 queries per niche at provision time
3. **Faster search** — reduce embedding dimensions, add DB index optimizations
4. **Pattern D (deferred messages)** — Ultravox deferred messages could let the agent speak while tool runs. Currently NOT implemented (see advanced-features-plan.md).

## Quick win
Check if `defaultReaction` is set on the queryKnowledge tool in `buildKnowledgeTools()`. If not, add it: "hmm let me look that up for you..."

## Files
- `src/lib/ultravox.ts` — `buildKnowledgeTools()` — check `defaultReaction` field
- `src/app/api/knowledge/[slug]/query/route.ts` — query latency

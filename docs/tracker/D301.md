# D301 — 29 Prompt Variables Locked Post-Onboarding

## Problem
The variable registry (D283a) documents 39 prompt variables. Only 10 are editable post-onboarding via settings cards/patchers. The other 29 (TRIAGE_DEEP, FIRST_INFO_QUESTION, COMPLETION_FIELDS, INDUSTRY, FORBIDDEN_EXTRA, FILTER_EXTRA, etc.) are locked at onboarding time.

The slot regenerator (D283c) makes these technically editable — `regenerateSlot()` can rebuild any section from current DB state. But there's no UI path and no API path for users to edit these 29 variables after onboarding.

## Root Cause: Niche Intake Fields Not Preserved
Many locked variables come from niche-specific intake fields (niche_emergency, niche_clientType, niche_pricingModel, etc.) that are consumed once by `buildSlotContext()` during onboarding. Their effects are baked into the prompt but the raw intake values are NOT stored to the DB.

This means `regenerateSlot()` can only reproduce the niche-default version of these variables — not the onboarding-customized version. A client who chose `niche_clientType: 'residential'` during onboarding will get the default `'plumbing company'` (not `'residential plumbing company'`) if their prompt is regenerated.

## Fix Path
1. **Phase 6 (D278 Agent Brain):** Expose all 39 variables as labeled, editable fields on the Agent Brain page
2. **Phase 6 (D280 recomposePrompt):** When a variable is edited, `recomposePrompt()` rebuilds the entire prompt from current DB state
3. **Niche intake field preservation:** Store niche-specific intake values to `niche_custom_variables` during onboarding so they survive to the DB and can be regenerated

## Affected Variables (29 locked)
INDUSTRY, CITY, LOCATION_STRING, PERSONALITY_LINE, GREETING_LINE, CLOSING_LINE, PRIMARY_GOAL, COMPLETION_FIELDS, CLOSE_ACTION, PRIMARY_CALL_REASON, FIRST_INFO_QUESTION, INFO_TO_COLLECT, INFO_LABEL, SERVICE_TIMING_PHRASE, MOBILE_POLICY, WEEKEND_POLICY, TRIAGE_DEEP, FILTER_EXTRA, NICHE_EXAMPLES, INFO_FLOW_OVERRIDE, CLOSING_OVERRIDE, SERVICES_NOT_OFFERED, INSURANCE_STATUS, INSURANCE_DETAIL, URGENCY_KEYWORDS, FORBIDDEN_EXTRA, SERVICE_APPOINTMENT_TYPE, PRICING_POLICY, UNKNOWN_ANSWER_BEHAVIOR

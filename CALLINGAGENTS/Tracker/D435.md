# D435 — Dental niche: niche-defaults exist but prompt-slots not implemented

**Status:** open
**Priority:** P1
**Filed:** 2026-04-06

## Problem

`niche-defaults.ts` has a dental entry (pain triage, new vs existing patient, insurance handling) but `prompt-slots.ts` has no dental-specific block. The dental prompt uses generic slot composition — no TRIAGE_DEEP rendering, no pain escalation logic, no insurance language injection. A dental client gets a shallow agent that collects name + reason but can't ask "is this for an existing patient?", triage pain level, or handle emergency dental calls.

## What's Already Built

In `niche-defaults.ts`:
- `INDUSTRY`: 'dental office'
- Emergency pain triage rules (severe pain → [URGENT])
- New vs existing patient routing
- Insurance handling: "we work with most dental insurance"
- LINGUISTIC_ANCHORS: dental terminology

## What's Missing

In `prompt-slots.ts`:
- No `case 'dental':` block in `buildConversationFlow()`
- No TRIAGE_DEEP rendering for pain level / urgency
- No new vs existing patient branch
- No emergency dental escalation path

## Fix

Follow the PM niche pattern:
1. NLM research query → dental caller types, guardrails, FORBIDDEN_EXTRA
2. Add `dental` block to `prompt-slots.ts` `buildConversationFlow()`
3. Update `niche-defaults.ts` dental entry with full TRIAGE_DEEP decision tree
4. Build `niches/dental.tsx` onboarding component (services chip selector, insurance toggle)
5. 3 test calls minimum
6. Verify Supabase row has correct `niche_custom_variables`

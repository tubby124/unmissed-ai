# D269 — Knowledge base as primary info source

## Summary
Move all factual business information out of the system prompt and into pgvector. The prompt gets a 1-line instruction: "Use your queryKnowledge tool to answer factual questions about this business." This replaces 1-3K chars of inline FAQ/knowledge per prompt.

## Why
Currently, business facts are duplicated:
1. Inline in the prompt as `PRODUCT KNOWLEDGE BASE` section (D265)
2. In pgvector as `knowledge_chunks` with `status='approved'`

The prompt copy is stale the moment the user updates their FAQ or knowledge. The pgvector copy is the runtime truth (queried via `queryKnowledge` tool). Removing the prompt copy eliminates duplication and saves 1-3K chars.

## Implementation
- Paired with D265 (remove PRODUCT KNOWLEDGE BASE)
- `buildKnowledgeBase()` slot returns:
  - If `knowledge_backend='pgvector'` AND chunks exist: 1-line queryKnowledge instruction
  - If `knowledge_backend='pgvector'` but no chunks: empty string
  - Else (no pgvector): current inline FAQ block (backward compat)
- Feature-gated on `knowledge_backend` field — safe rollback by toggling field

## Dependencies
- **Requires:** D274 (named slots for clean extraction)
- **Pairs with:** D265 (remove hardcoded KB), D268 (minimal base)
- **Enables:** D270 (frequent KB query → auto-suggest FAQ), D271 (PDF upload)

## Key Files
- `src/lib/prompt-slots.ts` — buildKnowledgeBase() slot function
- `src/lib/prompt-helpers.ts` — remove `buildNicheFaqDefaults()` (200+ lines)
- `src/lib/prompt-builder.ts` — remove KB replacement logic

## Risk
MEDIUM — clients without pgvector keep inline FAQ (backward compat). Clients with pgvector already have queryKnowledge tool working. Main risk: ensuring all 4 active clients have sufficient approved chunks before removing inline fallback.

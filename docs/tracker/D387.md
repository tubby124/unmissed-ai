# D387 — Chip/Pill Toggle UI Component for Onboarding

## Problem
All niche step files use raw HTML `<input type="radio">` and `<input type="checkbox">` with no styling. No shared design language across niche steps. Auto-glass has a 2-column checkbox grid (better), plumbing has plain radio labels, property-management mixes styles. Looks like a 2019 form.

NLM research: chips/pill toggles with toggle state reduce cognitive load vs radio/checkbox because:
- Visual state is immediately obvious (filled = selected, ghost = not)
- Can be tapped on mobile without tiny input hitbox
- Groups related options spatially

## What Needs to Be Done

### 1. Create `ChipSelector` component
`src/components/onboard/ChipSelector.tsx`:

**Single-select (replaces radio groups):**
```tsx
<ChipSelector
  options={[{ value: 'yes_24_7', label: '24/7 emergencies' }, ...]}
  value={selected}
  onChange={(v) => onChange('emergency', v)}
/>
```

**Multi-select (replaces checkbox grids):**
```tsx
<ChipSelector
  multi
  options={SERVICES}
  value={selectedServices}
  onChange={(vals) => onChange('services', vals)}
/>
```

**Visual spec:**
- Unselected: `border border-border bg-card text-muted-foreground hover:border-indigo-400`
- Selected: `border border-indigo-600 bg-indigo-50 dark:bg-indigo-950 text-indigo-700 font-medium`
- Shape: `rounded-full px-3.5 py-1.5 text-sm cursor-pointer transition-colors`
- Multi-select: show checkmark icon on selected chips
- Single-select: show filled dot or solid background

### 2. Replace raw inputs across all niche files
Priority order (by niche in production):
1. `auto-glass.tsx` — services grid + insurance + mobile service → chips
2. `plumbing.tsx` — emergency + clientType → chips; service area stays as Input
3. `property-management.tsx` — propertyType + unitCount → chips
4. All others — same pattern

### 3. Design rule
- **Single choice = chip single-select** (not radio group)
- **Multiple choice = chip multi-select** (not checkbox grid)
- **Free text = Input** (service area, custom additions)
- **Long text = Textarea** (maintenance contacts, tenant roster — keep as-is)

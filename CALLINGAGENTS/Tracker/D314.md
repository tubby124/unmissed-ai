# D314 — Unified Dashboard — Kill First-Visit Branch

## Problem

First-visit trial users saw `TrialActiveSection` — a separate, simplified onboarding view. Returning trial users and paid users saw `UnifiedHomeSection` (the full dashboard with stats, config strip, knowledge tiles, etc).

This meant:
- First-time trial users didn't see the real product on first login
- Two separate components to maintain for essentially the same page
- Trial users couldn't see capabilities (locked or available) until their second visit

## Research (Sonar Pro — 2026-04-01)

Ran 2 Perplexity Sonar Pro queries on SaaS trial dashboard UX best practices:

**Key findings:**
1. **Full dashboard with subtle upgrade hints converts better than simplified views** — users need to see what the product does to want to pay for it
2. **Feature teasing > feature gating** — soft "Upgrade" / "Set up" pills outperform hard lock icons (triggers psychological "reactance")
3. **Empty states must guide, not be blank** — setup progress bars, next-action strips, and contextual CTAs fill the void for users with no data yet
4. **For voice AI specifically:** the agent is already provisioned, so first action should be "test your agent" — which `UnifiedHomeSection` already has via `TestCallCard` and the call-me widget

## Decision

Kill the `TrialActiveSection` first-visit branch. ALL non-expired users (trial first-visit, trial returning, paid-awaiting, paid-ready) now see `UnifiedHomeSection`.

The trial-specific elements INSIDE `UnifiedHomeSection` are preserved:
- Trial countdown pill
- "Get my number" upgrade CTA
- "Call me through your agent" phone widget
- Trial expiry warning (≤3 days)

QuickConfigStrip already uses a "tease" pattern (pills show "Set up" / "Upgrade" / "Off" / "Connected") — no hard lock icons. This aligns with the research finding that teasing > gating.

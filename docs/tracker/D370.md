# D370 — Agent Variables visible on Overview

## Problem
Agent variables (callback person name, business name, agent name, etc.) are currently only editable under Knowledge tab → Agent Variables. Users should be able to see and quickly edit these from the Overview page.

## Context
Discovered during mushroom dispensary onboarding — user didn't realize "Emon" was set as the callback person until checking Settings. If it were visible on Overview, they'd have caught it immediately.

## Proposed Solution
Add a compact "Agent Identity" or "Agent Variables" card to the Overview page (likely left column near KnowledgeInlineTile) showing key variables with inline edit capability.

## Key Files
- `src/components/dashboard/home/UnifiedHomeSection.tsx` — Overview layout
- `src/components/dashboard/home/KnowledgeInlineTile.tsx` — sibling tile
- `src/app/api/dashboard/variables/route.ts` — variable edit API (D303)

## Acceptance Criteria
- [ ] Key variables (agent name, callback person, business name) visible on Overview
- [ ] Inline edit or deep-link to edit
- [ ] Updates trigger prompt recompose if needed

# Promote-on-edit pattern (the actual systemic fix)

## Origin (2026-05-05)

Brian's drift root cause: 8 custom examples (A-H) live in `clients.system_prompt` raw text but have no corresponding DB column. Slot pipeline can't reproduce them on recompose. This pattern repeats every time someone polishes a prompt via `/build-prompt-ui`, the dashboard prompt editor, or direct SQL. Each tuning session creates more loose drift. **D451 (dialog gate) and D452 (drift detection) treat the symptom; this is the structural fix.**

## Build

### Phase A — Schema additions

Add structured override columns to `clients`:

```sql
ALTER TABLE clients
  ADD COLUMN inline_examples_override jsonb,           -- replaces niche default examples
  ADD COLUMN voice_anchors_override text,              -- replaces niche default voice anchors
  ADD COLUMN tone_overrides jsonb,                     -- per-section tone tweaks
  ADD COLUMN section_overrides jsonb;                  -- catch-all: { section_name: "override text" }
```

Slot context (`SlotContext` in [src/lib/prompt-slots.ts](src/lib/prompt-slots.ts)) reads each override before falling back to niche default. Recompose preserves overrides by construction.

### Phase B — `/build-prompt-ui` outputs structured

When the skill produces a polished prompt:
1. Extract custom examples → write to `inline_examples_override`
2. Extract voice anchors → write to `voice_anchors_override`
3. Extract any prompt-section tweaks → write to `section_overrides`
4. Run recompose → verify output matches the polished version
5. NEVER write to `clients.system_prompt` directly

### Phase C — Dashboard prompt editor

The "Edit Prompt" UI currently lets admins write raw text to `system_prompt`. Replace with section-by-section editor that writes to `section_overrides[section_name]`. Raw text mode hidden behind admin-with-warning flag.

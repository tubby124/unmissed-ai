# D358 — Inline variable editing from Overview/Knowledge

## Status: DONE ✅ (2026-04-01)

## What Was Built

### GET /api/dashboard/variables
New endpoint that resolves current variable values for the authenticated client. Combines niche defaults → niche_custom_variables → DB column overrides using `clientRowToIntake()` + `buildSlotContext()`. Returns all 39 variables with their resolved values and metadata. Supports admin `?client_id=` override.

### PATCH /api/dashboard/variables — admin support
Added `body.client_id` support for admin users (matching the pattern from settings PATCH). Existing single-variable save + slot regeneration unchanged.

### PromptVariablesCard component
New settings card at Settings → Agent tab → Knowledge section. Features:
- 5 collapsible groups: Identity, Opening & closing, Services & knowledge, Call routing, Guardrails
- Shows resolved variable values with filled/total counter per group
- Inline editing: click Edit → text input (short) or textarea (multiline) → Save
- Each save calls `PATCH /api/dashboard/variables` which writes to the correct storage (DB column or niche_custom_variables JSONB) and regenerates only affected prompt slots
- Prompt change notification to parent via `onPromptChange` callback
- Dedicated-card variables (hours, transfer, etc.) shown as read-only with "via Settings" tag

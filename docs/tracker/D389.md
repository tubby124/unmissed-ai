# D389 — In-Flow "Aha Moment": Hear Your Agent Before Going Live

## Problem
The biggest conversion lever in voice AI onboarding is when the prospect hears the AI work for the first time. Currently this happens AFTER onboarding (on the status page via WebRTC). That's too late — the user has already committed.

NLM Q1/Q3/Q4 all converge on this: "Experiencing this live conversation is what instantly transforms a skeptical prospect into a believer." Platforms that get the aha moment IN the flow see dramatically higher trial-to-paid conversion.

## What Needs to Be Done

### Option A — On the Launch step, before "Go Live" (recommended)
Add to `step6-activate.tsx` above the "Go Live" button:
```
┌─────────────────────────────────────────┐
│  🎧 Hear your agent before going live   │
│                                         │
│  Your agent is already built. Call it:  │
│  [📞 Call via browser]  [📋 Copy number] │
│                                         │
│  "Hi, thank you for calling Prairie     │
│   Plumbing. This is Jacqueline..."      │
└─────────────────────────────────────────`
```

Show the WebRTC test panel inline, collapsed by default. "Test your agent" expands it.

### Option B — Dedicated step between Setup and Plan
Add a "Hear it" step with a phone icon and the browser WebRTC call. Plan picker comes after — once they've heard it, they're sold on upgrading.

**Recommended order:** Business → Setup → Hear It → Plan → Launch

## Why This Works
1. Business owner hears their own agent name + business name immediately
2. Validates that PERSONA sandwich works (sounds like their business, not generic)
3. Builds confidence before they enter payment info (trial) or commit
4. Agent actually works at this point — GBP data + niche defaults + Haiku TRIAGE_DEEP already generated in background

## MVP Implementation
Reuse existing `AgentPreviewCard.tsx` or the WebRTC component from the status page. Provision needs to have happened first — so this works best at the Launch step AFTER the agent is provisioned, OR as a "demo mode" against the standard niche demo agent before provisioning.

**Easier path:** At the Launch step, show "Preview your agent" with a link to call the niche demo agent (Zara / existing demo) with their business name injected. Not their actual provisioned agent — just a close-enough demo.

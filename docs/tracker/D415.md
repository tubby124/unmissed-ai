# D415 — Verify Telegram notification format includes PM context (P1/P2/P3 + unit + tenant + category)

**Status:** 🔴 Open — Needs verification
**Priority:** HIGH
**Date:** 2026-04-05

## Problem
Jade uses [P1 URGENT], [P2 URGENT], [P3 ROUTINE], and [SHOWING REQUEST] tags in her CLOSING_OVERRIDE. But it's unclear whether the Telegram notification that fires after the call actually surfaces these tags prominently along with the unit number, tenant name, and issue category.

A PM's Telegram message needs to read:
```
[P1 URGENT] 🚨 Unit 4A — John Smith
Issue: burst pipe (plumbing)
Phone: (416) 555-1234
Time: 2:14am
→ REQUIRES IMMEDIATE RESPONSE
```

Not just:
```
New call from (416) 555-1234 — left a message
```

## What to Check
1. What does the AI summary generated by OpenRouter look like for PM calls? Does it extract tags + unit + issue?
2. What does the Telegram message body look like for PM calls?
3. Is the [P1 URGENT] tag parsed and used to change the Telegram message style (🚨 prefix, urgent language)?

## Files to Check
- `src/lib/transcript-analysis.ts` — AI summary generation
- The webhook handler or notification sender — look for where Telegram is called with call summary
- `src/app/api/webhook/[slug]/completed/` route — where notifications fire

## If Tags Are Not Parsed
Add PM-specific notification formatting:
- Parse `[P1 URGENT]`, `[P2 URGENT]`, `[P3 ROUTINE]`, `[SHOWING REQUEST]` from the AI summary
- Format the Telegram message body based on the tag
- P1 → 🚨 urgent prefix + "REQUIRES IMMEDIATE RESPONSE" footer
- SHOWING REQUEST → 🏠 prefix + showing times extracted from summary

# D385 — queryKnowledge circuit breaker — prompt-level fallback instruction

**Status:** ✅ Done
**Date:** 2026-04-04

## What
Updated `buildKnowledgeBaseSlot()` in `prompt-slots.ts` to include explicit circuit breaker instructions when knowledge base queries return empty or fail:

```
If queryKnowledge returns no results or an empty answer:
  say "I don't have that detail in front of me — I'll have [CLOSE_PERSON] follow up with you directly on that."
  Do NOT guess or fabricate an answer.

If the tool call fails or returns an error:
  say "I'm having a bit of trouble looking that up right now — I'll have [CLOSE_PERSON] follow up with you directly."
  Do NOT retry more than once. Do NOT make up information.
```

## Why
NLM Q3 (notebook 8d82f254) confirmed that without an explicit circuit breaker, agents hang, hallucinate, or expose technical errors to callers when KB queries return empty. The exact spoken phrases ensure the transition feels natural and the call doesn't drop.

This is a prompt-level instruction, not a code-level retry mechanism. The agent is told what to *say* when the tool returns empty — which is more reliable than catching errors in middleware.

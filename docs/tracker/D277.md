# D277 — Lag Root Cause Investigation (plumber-calgary-nw) — REMOVED

## Resolution
**REMOVED from active tracker 2026-03-31.** plumber-calgary-nw was built with the old monolithic prompt system. The prompt architecture refactor (D268 — minimal base prompt + dynamic sections) solves this structurally by shrinking prompts from 12-18K to 4-5K base. No separate investigation needed.

The 4 working clients (hasan-sharif, exp-realty, windshield-hub, urban-vibe) are NOT being redeployed. plumber-calgary-nw stays as a broken test case that validates the new architecture works.

## Original Investigation Findings (preserved for reference)

| Finding | Implication |
|---------|-------------|
| No hard token limit in Ultravox | Lag from responsiveness degradation, not cutoff |
| Recommended: under 6K chars for voice | 18.5K was 3x recommended — compression to 5.3K is within range |
| queryKnowledge for all facts | Knowledge tool with 0 hits may cause silent failed queries |
| Too many tools = tool selection overhead | Needs comparison with working agents |
| Duplicate services in contextData | 21 services (with dupes) adds unnecessary tokens |

### Likely contributing factors (ranked)
1. **Prompt size (18.5K chars)** — 3x recommended. Now compressed to 5.3K.
2. **Knowledge tool with 0 hits** — Round-trip latency per failed query.
3. **Tool count** — More tools = more tokens in descriptions.
4. **Duplicate services in context_data** — Unnecessary token overhead.

All of these are resolved by the prompt architecture refactor (D268 + D265 + D272 + D260).

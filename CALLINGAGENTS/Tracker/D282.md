# D282 — Business Name Change → Auto-Patch Prompt

## Problem
`business_name` is baked into the prompt at provision time. If user changes business name in settings, the prompt still says the old name. `patchBusinessName()` exists but was only recently wired — drift risk 6 in mutation contract.

## Fix Plan
Already partially implemented:
- `patchBusinessName()` in `prompt-patcher.ts` ✅
- Wired in `settings-patchers.ts` ✅
- But: only fires if `currentBusinessName` (old) is found in prompt text
- If prompt was manually edited, the old name might not match → silent skip

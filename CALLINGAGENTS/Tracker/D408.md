# D408 — VIP Contacts / Tenant Caller ID

## Problem
PM clients have no way to register known tenants by phone number. When a tenant calls, Jade has no way to greet them by name or pull their unit number automatically — requires tenant to identify themselves every time.

## Fix
- Add a `vip_contacts` table (or reuse existing if it exists) with `client_id`, `phone`, `name`, `unit_number`, `notes`
- On inbound call, cross-reference CALLER PHONE against vip_contacts
- If matched: inject `VIP CALLER: {name}, Unit {unit}` into callerContext
- Dashboard UI to add/edit/delete tenant contacts

## Files
- `src/lib/agent-context.ts` — inject VIP match into callerContext
- `src/app/api/dashboard/vip-contacts/` — already exists, check if PM-capable
- Dashboard component for tenant roster management

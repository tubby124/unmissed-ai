# D404 — Wire maintenanceContacts field into PM system prompt

**Status:** 🔴 Open
**Priority:** HIGH
**Date:** 2026-04-05

## Problem
`property-management.tsx` collects a `maintenanceContacts` freetext field during onboarding:
```
Plumbing: Mike (416-555-1234) — leaks, burst pipes
Electrical: Bob (416-555-5678) — power outage
```

But this field is never injected into the system prompt. Jade doesn't know who Mike is and can't route emergency calls to the right person.

The tenant roster (`tenantRoster`) IS injected → `context_data` with label `TENANTS`. Same pattern needed for maintenance contacts.

## Solution
In `intake-transform.ts`, add:
```ts
if (data.niche === 'property_management' && data.nicheAnswers?.maintenanceContacts) {
  // inject as injected_note or as a named section in the prompt
}
```

In `voicemail-prompt.ts` PM section, add a `MAINTENANCE_CONTACTS` block after the PM notes:
```
MAINTENANCE CONTACTS
{{MAINTENANCE_CONTACTS}}
Route emergency calls to the appropriate contact based on issue type.
```

## Files
- `src/lib/intake-transform.ts`
- `src/lib/prompt-niches/voicemail-prompt.ts`
- `src/lib/prompt-slots.ts` (add slot if using slot registry)

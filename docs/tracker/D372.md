# D372 — niche='other' generates bloated 19K char prompt

## Problem
When a client onboards with an unrecognized niche (falls to `niche='other'`), the generated system prompt is 18,999 chars — far over the 12K hard max. The Electric Avenue Mushroom Dispensary client triggered this.

## Root Cause
- Forbidden actions section has duplicate rules (rules 10-16 appear twice with different content — standard set + niche-specific "other" set)
- Generic template doesn't get the same compression/optimization as known niches
- 6 inline examples + full conversation flow with 6 intent categories adds significant bulk
- Knowledge section is nearly empty despite 13 business facts being available

## Impact
- Prompt exceeds `validatePrompt()` 12K limit
- Agent behavior degraded by noise/repetition in oversized prompt
- GLM-4.6 repetition loop risk increases with prompt bloat (per `memory/glm46-prompting-rules.md`)

## Fix Plan
1. Deduplicate forbidden action rules in `niche='other'` path
2. Compress generic template to match known-niche compression ratios (42-58% reduction achieved for known niches)
3. Consider whether `niche='other'` should use a minimal slot set instead of the full template
4. Validate that `buildPromptFromIntake()` respects the 12K limit for all niches

## Key Files
- `src/lib/prompt-slots.ts` — slot functions
- `src/lib/prompt-builder.ts` — `buildPromptFromIntake()`
- `src/lib/prompt-patcher.ts` — patchers

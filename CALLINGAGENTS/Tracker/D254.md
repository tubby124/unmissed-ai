## Implementation (2026-03-31) — DONE ✅

**Build verified: `✓ Compiled successfully in 6.2s`**

Post-onboarding complement to [[Tracker/D247]]. Owners can edit their "why do people call?" reasons and regenerate TRIAGE_DEEP from Settings > Agent > What It Can Do.

### Changes made
1. `src/components/dashboard/settings/CallRoutingCard.tsx` — new settings card (~170 lines)
   - 3 text inputs for caller reasons (niche-aware placeholders)
   - "Set up routing" / "Regenerate" button → POST `/api/onboard/infer-niche` with `{businessName, knownNiche, callerReasons}`
   - Merges TRIAGE_DEEP + `_caller_reasons` into `niche_custom_variables` → PATCH `/api/dashboard/settings`
   - Shows "Routing active" badge when TRIAGE_DEEP is populated
   - Reasons stored as `_caller_reasons: JSON.stringify(filled)` for pre-population on next open
2. `src/components/dashboard/settings/AgentTab.tsx` — wired CallRoutingCard in "What It Can Do" section

### Mutation Class
`DB_ONLY` — `niche_custom_variables` is JSONB. TRIAGE_DEEP feeds `buildPromptFromIntake()` at provision time, not the live `system_prompt`. To take effect on the live agent, owner must also trigger a prompt regeneration (tracked as D256).

### Known gap (D256)
TRIAGE_DEEP changes do NOT automatically sync to the live Ultravox agent. Requires prompt regeneration + `/prompt-deploy`. Card shows a note about this.

# D321 — Step 3 and Step 5 duplicate FAQ collection

## Problem
Two steps collect FAQ/knowledge data via different mechanisms:

**Step 3** — `callerFaqText` (freeform textarea):
- "What questions do callers ask most?"
- Provision route sends this to Haiku → extracts Q&A pairs → writes to `extra_qa`

**Step 5** — `faqPairs` (structured Q&A cards):
- Niche-specific default questions with answer fields
- Written to `niche_faq_pairs` → knowledge seeding

User enters FAQ info twice in two different formats across two different steps.

## Fix
Consolidate into Step 5 only:
- Remove `callerFaqText` from Step 3
- Step 5 keeps the structured `faqPairs` editor (better UX, answers go directly to KB)
- If we want the freeform "paste your FAQs" option, add it to Step 5 as an alternative input mode

## Dependency
- D318 (step 3 trim) removes callerFaqText as part of the cleanup

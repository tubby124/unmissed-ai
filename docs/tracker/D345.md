# D345 — Intelligence seed loading indicator

## Problem
`fireIntelligenceGeneration()` runs silently in the background after GBP confirm. Takes ~10 seconds (Haiku via OpenRouter). User has zero visibility that anything is happening. If they move fast through plan selection, they could reach the Launch step before the seed arrives and see niche defaults instead of personalized intelligence.

## Observed during live test (2026-04-01)
- GBP confirmed → intelligence fired → user picked plan → reached Launch step
- In this test the seed arrived in time (~10s) and the AgentIntelligenceCard rendered correctly
- But a fast user or slow API would miss it entirely — no error, no indication, just falls back to generic defaults

## Options (pick one)
1. **Subtle loading pill on GBP card**: "Configuring your agent..." → "Agent configured ✓" — non-blocking, just informational
2. **Spinning orb on Launch step** if seed hasn't arrived: "Personalizing your agent..." with 5-second max wait before showing defaults
3. **AgentIntelligenceCard shows skeleton state** instead of being hidden when seed is pending — shimmer animation, then fills in when data arrives
4. **Progress toast** at bottom: "AI is learning about your business..." → "Done — 5 intents configured"

## Recommendation
Option 3 (skeleton card) + Option 1 (pill). The card should always render on the Launch step — show skeleton if pending, fill in when ready. The GBP card gets a small status pill so the user knows something smart is happening.

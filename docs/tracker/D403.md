# D403 — Unblock P1 emergency transfer in PM niche

**Status:** ✅ Done (already fixed in session 2026-04-05)
**Priority:** CRITICAL
**Date:** 2026-04-05

## What Was Fixed
`niche-defaults.ts` PM section FORBIDDEN_EXTRA was already updated to allow P1 transfers:
> "NEVER pretend to transfer or put someone on hold. This is a callback-only service."

This overrides the global transfer slot even when the PM client has `after_hours_emergency_phone` set and `caps.transferCalls = true`. A 2am burst pipe call CANNOT transfer — Jade just takes a message. That's the wrong behavior for P1 emergencies and the #1 reason a PM company won't trust this product.

## Solution
Remove the blanket transfer ban from PM FORBIDDEN_EXTRA. Replace with:
```
"NEVER transfer except for confirmed P1 emergencies (flooding, no heat, gas smell, fire, break-in, burst pipe).
For all other calls — billing, routine maintenance, prospects — route to callback only.
NEVER promise to put someone on hold."
```

P1 transfer only fires when `transferCall` tool is registered (i.e., `forwarding_number` or `after_hours_emergency_phone` is set). No tool = no transfer = safe.

## Setup Required (for test)
1. `after_hours_emergency_phone` = cell number
2. `after_hours_behavior` = `route_emergency`
3. Fix FORBIDDEN_EXTRA (this ticket)
4. Rebuild prompt → test call: "hi my pipe just burst"

# D441 — Zara homepage demo prompt rewrite + identity drift fix

## Problem
Zara felt robotic and strict on the homepage. 11,991-char prompt (99% of 12K hard cap), 14 NEVER rules at top, rigid "<25 words" + "MUST use casual pattern in EVERY response" forcing functions. DB had `agent_name='Aria'` but prompt body + TalkToAgentWidget said "Zara" — drift bug.

## Solution
Full prompt rewrite. Target 7,500-8,500 chars; landed at 10,844 (over target, well under hard cap). Cut: EMERGENCY OVERRIDE, RETURNING CALLER, strict <25-words, MUST-casual rigidity, 9 NEVER rules. Add: capability-signal triage, energy-match, confirm-back, skip-step, single-okay-not-goodbye, no-hollow-affirmations. DB drift fixed via `update clients set agent_name='Zara' where slug='unmissed-demo'`.

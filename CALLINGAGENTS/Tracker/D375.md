# D375 — Backfill recompose: code fixes don't heal existing agents

## Problem
Every prompt improvement only applies to new onboardings. Existing clients accumulate bugs forever.
Red Swan: CLOSE_PERSON="Red", greeting says "AI assistant", hours say "23pm" — all fixed in code, none applied to DB.

## Fix
Admin "Rebuild Prompt" per-client button that runs `recomposePrompt()` with current slot logic + existing DB variables.
OR: bulk "Recompose All" for admin — iterate all clients, run recomposePrompt(), sync to Ultravox.

## Files
- `src/app/api/admin/recompose-all/route.ts` — new endpoint
- `src/app/dashboard/admin/` — admin UI trigger
- `src/lib/prompt-slots.ts` — already has recomposePrompt()

# D326 — Paste-and-parse discards structured service data (price, description, duration)

## Problem
The freeform paste feature (D126) uses Haiku to extract structured service data:
```json
{"name": "Oil change", "description": "Full synthetic...", "price": "from $65", "duration_mins": 30}
```

But at provision time (`provision/trial/route.ts` lines 217-219), only the service NAME is used. Description, price, and duration_mins are all set to empty strings.

The structured `parsedServiceDrafts` data stored in onboarding state never reaches the provision route.

## Data flow (current)
```
User pastes text → POST /api/onboard/parse-services → Haiku extracts {name, description, price, duration}
                 → names added to selectedServices[] ← this is all that flows through
                 → parsedServiceDrafts stored in local state only ← NEVER read by provision
```

At provision:
```
selectedServices = ["Oil change", "Brake repairs"]
→ client_services rows = {name: "Oil change", description: '', price: '', duration_mins: null}
```

## Fix
In `provision/trial/route.ts`, when building `client_services` rows, check if `data.parsedServiceDrafts` has a matching entry and merge the structured data:

```ts
const draftsMap = new Map(
  (data.parsedServiceDrafts ?? []).map(d => [d.name, d])
);
const serviceRows = selectedServices
  .map((name, idx) => {
    const draft = draftsMap.get(name);
    return {
      client_id: clientId,
      name: name.trim().slice(0, 200),
      description: draft?.description ?? '',
      price: draft?.price ?? '',
      duration_mins: draft?.duration_mins ?? null,
      ...
    };
  });
```

Also wire `parsedServiceDrafts` into `toIntakePayload()` for the `service_catalog` injection used by `buildPromptFromIntake()`.

## Files
- `src/app/api/provision/trial/route.ts` — lines 209-219 (service_catalog injection) and 473-498 (client_services insert)
- `src/lib/intake-transform.ts` — `toIntakePayload()` doesn't read `parsedServiceDrafts`

## Size
Small — merge structured data into existing rows. ~15 lines changed.

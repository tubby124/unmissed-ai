# D343 — Add order read-back to restaurant TRIAGE_DEEP

## Problem (Sonar Pro validated)
Restaurant AI agents that don't read back orders have significantly higher error rates. Callers fear wrong orders when there's no confirmation. Read-back is the #2 success factor after menu knowledge (Certus AI, Loman AI both do this).

## Current behavior
FOOD_ORDER intent in TRIAGE_DEEP says:
```
→ Collect: items + sizes, pickup or delivery, name, pickup time or delivery address
→ Outcome: take order details for Sal
```
No read-back step.

## Fix
Add to FOOD_ORDER outcome in the intelligence seed prompt template:
```
→ Before closing: read back the full order: "so that's a [size] [item], [size] [item], for [pickup/delivery] under [name] — did I get that right?"
→ Outcome: confirm order → take order details for Sal
```

## Files
- `src/app/api/onboard/generate-agent-intelligence/route.ts` — update the TRIAGE_DEEP example in the Haiku prompt to include read-back
- `tests/promptfoo/sals-ny-pizza-test.yaml` — add test: multi-item order → agent reads back before closing

# D243 — TRIAGE Section Rewrite: Intent-First Routing

## Problem
Current TRIAGE template asks for caller details first, then routes. Backwards.
GLM-4.6 reads top-first → defaults to info-bot regardless of mode because PRIMARY GOAL is at the end.

## Fix
In `src/lib/prompt-builder.ts`, rewrite TRIAGE template structure:
```
STEP 1: Identify intent from first sentence
  → "book/appointment/schedule" → BOOKING FLOW
  → "price/cost/quote/how much" → QUOTE FLOW
  → "speak to someone/transfer/manager" → TRANSFER FLOW
  → "info/question/hours/location" → INFO FLOW
STEP 2: Confirm intent in one sentence
STEP 3: Execute the appropriate flow
```

Also: move PRIMARY GOAL block ABOVE TRIAGE in the template.

Must keep in sync: `PROVISIONING/app/prompt_builder.py` (see `memory/feedback-deploy-tool-parity.md`)

## Files
- `src/lib/prompt-builder.ts` — TRIAGE + PRIMARY GOAL sections
- `PROVISIONING/app/prompt_builder.py` — parallel Python deploy tool

- caller opens with "I want to book" → agent goes directly to booking flow (no name interrogation first)
- caller opens with "how much?" → price info without name first
- D245 passes for all 4 active clients

# D276 — Booking Flow Auto-Updates Call Flow

## Problem
`patchCalendarBlock()` appends/removes the CALENDAR BOOKING FLOW section when `booking_enabled` toggles, but the TRIAGE and GOAL sections don't adapt. The agent's triage questions and closing don't change to reflect booking availability.

## Solution
In `settings/route.ts`, after `booking_enabled` changes (step 8c), call `regenerateSlots(clientId, ['conversation_flow', 'goal'])`. This makes the triage questions and closing adapt to whether calendar booking is available.

Skips if `niche_custom_variables` already triggered a full regen (no duplicate work).

## Key files
- `src/app/api/dashboard/settings/route.ts` — step 8c wiring

## Status
**DONE** (2026-03-31) — Phase 6 Wave 1. Only fires on slot-composed prompts (new clients). Old clients use patchers only.

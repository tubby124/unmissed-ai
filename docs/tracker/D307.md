# D307 — Recompose Warning UX

Confirmation + diff preview before recompose runs. Ships WITH the recompose button — never a recompose without a warning.

## What This Covers
- User clicks "Regenerate prompt" → modal shows diff (current vs proposed)
- Clear explanation: "This will rebuild your agent's prompt from your current settings"
- Diff view: red/green showing what changes
- "Confirm" / "Cancel" buttons
- Uses D305 backend (dry-run preview API)

## Dependencies
- D305 backend ✅ DONE (preview API exists)
- D305 frontend (diff UI component) — ships together with this
- D280 ✅ DONE (recomposePrompt exists)

## Files
- `POST /api/dashboard/variables/preview` — backend exists
- New: `src/components/dashboard/RecomposeConfirmModal.tsx`

## Acceptance
- [ ] Modal shows before any recompose
- [ ] Diff is readable (red = removed, green = added)
- [ ] Cancel returns to previous state with no changes
- [ ] Confirm triggers recompose and shows success/error

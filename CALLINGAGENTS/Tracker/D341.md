# D341 — PromptVariablesCard (was D283b)

**Status:** ✅ DONE 2026-04-24 — Shipped as `AgentRoutesOnCard.tsx` on the Wave 2 branch. Read-only preview of GREETING_LINE / URGENCY_KEYWORDS / FORBIDDEN_EXTRA / CLOSE_PERSON from `clients.niche_custom_variables` (sourced via `/api/dashboard/home`, which now returns `nicheCustomVariables`). Every row deep-links into Settings > Agent tab (#greeting / #urgency / #safety / #close-person). Edit surface stays in Settings — this card avoids duplicate edit surfaces while improving discoverability on Overview.


## Problem
The intelligence seed (TRIAGE_DEEP, GREETING_LINE, URGENCY_KEYWORDS, FORBIDDEN_EXTRA) is generated during onboarding and baked into the prompt. The owner NEVER sees it again. They can't:
- See how their agent routes callers
- Edit the greeting line
- Add/remove urgency triggers
- Modify the NEVER rules

## What exists (backend)
- `PATCH /api/dashboard/variables` — accepts `{ variableKey, value }`, writes to correct storage, regenerates affected slots
- `POST /api/dashboard/variables/preview` — dry-run: shows what prompt would look like after change
- `src/lib/prompt-variable-registry.ts` — 39 registered variables with metadata (label, description, dbField, locked, affectedSlots)
- `src/lib/slot-regenerator.ts` — regenerates only the affected prompt sections

## What's missing (frontend)
A settings card (or section on the Knowledge page) that:
1. Reads `clients.niche_custom_variables` to get current values
2. Displays key variables in a human-readable format:
   - **Opening line** (GREETING_LINE) — editable text
   - **Caller routing** (TRIAGE_DEEP) — read-only or section-editable
   - **Urgency triggers** (URGENCY_KEYWORDS) — tag/chip editor
   - **Safety rules** (FORBIDDEN_EXTRA) — list editor
   - **Close person** (CLOSE_PERSON) — text input
3. On save: calls `PATCH /api/dashboard/variables` per field
4. Shows "Preview" button that calls the preview endpoint

## Design
Run through `/ui-ux-pro-max` before building. Consider:
- Could live on Settings → Agent tab
- Or as a section on the Knowledge page
- Expandable by default (most owners won't touch it, but power users want to)
- "AI-configured" badge like the onboarding card

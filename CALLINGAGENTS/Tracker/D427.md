# D427 — Resend sandbox mode blocks PM email notifications

**Status:** open
**Priority:** P0
**Filed:** 2026-04-06

## Problem
Resend is in sandbox mode on Railway. `notifyPm()` in maintenance-request/route.ts calls Resend to send P1 URGENT emails, but sandbox mode only delivers to verified addresses. New PM clients get Telegram notifications but their P1 email is silently dropped. notification_logs shows status='sent' which is misleading.

## Root Cause
`RESEND_API_KEY` on Railway points to a sandbox account. No domain verification on notifications@unmissed.ai.

## Fix
(1) Verify notifications@unmissed.ai domain in Resend dashboard, (2) Upgrade Resend account to production mode, (3) Update RESEND_API_KEY on Railway with production key. No code changes needed.

## Notes
This is a launch blocker for the PM niche. Every PM client onboarded until this is fixed will have broken P1 email alerts.

# D295 — Multi-Source Knowledge Ingestion (dashboard)

## Problem
Knowledge input is limited to one website scrape + manual FAQ. Users can't add multiple websites, upload PDFs from dashboard easily, or connect GBP after onboarding.

## Vision
Dashboard "Knowledge Sources" panel:
- Add website URL (multiple) → scrape + extract → knowledge_chunks
- Upload PDF/Word doc → extract → knowledge_chunks  
- Connect GBP → sync hours, services, reviews → knowledge_chunks
- Manual entry → type facts/FAQ directly
- "Learned from calls" → auto-detected facts from call transcripts

All sources visible, rescannable, removable. Each source tracked with last-sync date and fact count.

## Technical Notes
- PDF upload pipeline already exists (knowledge upload routes)
- Website scraper exists (D246)
- GBP import designed (D291) — extend from onboarding-only to dashboard
- Need: multi-URL support (currently one website_url column)
- Need: knowledge_chunks.source tracking per URL/doc (source_url column exists)
- Need: scheduled re-scan for website sources (cron)

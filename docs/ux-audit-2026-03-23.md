### Typography Scale (5 sizes only)

| Name     | Size  | Weight | Line-height | Use for                    |
|----------|-------|--------|-------------|----------------------------|
| heading  | 20px  | 600    | 1.3         | Page titles, hero numbers  |
| subhead  | 16px  | 600    | 1.4         | Card titles, section heads |
| body     | 14px  | 400    | 1.5         | All body text, nav items   |
| label    | 12px  | 500    | 1.4         | Labels, badges, captions   |
| micro    | 11px  | 500    | 1.3         | Timestamps, counters       |

**BANNED:** text-[9px], text-[10px], text-3xl, text-2xl, text-lg, text-base (use the 5 above)

### Spacing Scale (strict 8px grid)

| Token | px  | Tailwind | Use for                    |
|-------|-----|----------|----------------------------|
| 1     | 4px | p-1      | Icon padding, tight gaps   |
| 2     | 8px | p-2      | Intra-element spacing      |
| 3     | 12px| p-3      | Element padding            |
| 4     | 16px| p-4      | Card padding, section gap  |
| 5     | 20px| p-5      | Section padding            |
| 6     | 24px| p-6      | Page padding               |
| 8     | 32px| p-8      | Large section spacing      |

**Rule:** Cards always `p-4` on mobile, `p-5` on desktop. Never `p-3 sm:p-6` (too big a jump).

## PART 4: INFORMATION ARCHITECTURE

### What useful info is MISSING from the dashboard

1. **No call quality trend** — just a single number. Show a sparkline or weekly trend.
2. **No "what your agent knows" summary** — user can't see what the agent will say about their business
3. **No "last trained" indicator** — user doesn't know if their agent is up to date
4. **No response time metric** — how fast does the agent pick up? Average ring time.
5. **No missed call rate** — percentage of calls that went unanswered
6. **No caller geography** — where are calls coming from?
7. **No peak hours** — when do most calls happen? (useful for SMB owners)
8. **No "what callers ask about most"** — top 5 topics from call transcripts
9. **No money saved estimate** — "Your agent handled X calls this month, saving you ~$Y in receptionist costs"

### What info should be REMOVED or HIDDEN

1. **Avg Quality score** — meaningless number (4.1 out of what?) without context
2. **Sentiment** column in call list — takes space, rarely actionable
3. **"Hot Leads" count for 0** — depressing for new users (already fixed in Wave 1)
4. **Bookings count for SMBs without booking** — irrelevant metric

### What info should be REORGANIZED

1. **Merge /dashboard and /dashboard/calls** — one home page, not two
2. **Hero card → simpler** — just agent name + status + minutes bar. Move call count into stats.
3. **Stats grid → contextual** — show different stats based on niche (real_estate vs auto_glass vs voicemail)
4. **Recent calls → richer** — show caller name if available, call duration prominently, sentiment as color dot not text

## PART 5: LIGHT MODE SPECIFIC FIXES

### Current Light Mode Problems
1. Cards blend into page background (both white)
2. Text hierarchy not visible enough (text-2 and text-3 too similar)
3. Borders too light (almost invisible)
4. No depth perception — everything flat

### Light Mode Fixes
1. Page bg: `#f7f8fa` (not pure white — gives cards contrast)
2. Card bg: `#ffffff` (pure white stands out on gray page)
3. Border: `#e2e5e9` (slightly stronger than current)
4. text-2: `#5f6368` (Google's secondary text — proven readable)
5. text-3: `#9aa0a6` (distinctly different from text-2)
6. Add `shadow-xs` to cards in light mode (1px shadow for depth)
7. Active nav: use accent-tint background (soft indigo wash)

## PART 7: MOBILE-SPECIFIC AUDIT

### What Works
- MobileNav hamburger menu is functional
- Touch targets are mostly 44px+
- Cards stack vertically
- Call list is scrollable

### What Needs Fixing
1. **Font sizes** — `text-[9px]` and `text-[10px]` are UNREADABLE on mobile. Minimum 11px.
2. **Card padding** — `p-3` on mobile is tight. Use `p-4` minimum.
3. **Bottom nav** — Consider fixed bottom nav bar (4-5 items) instead of hamburger. More Apple-like. SMB owners check dashboards in 10-second bursts — hamburger adds friction.
4. **AgentTestCard on mobile** — the "Start Test Call" button should be full-width
5. **Stats grid** — `grid-cols-1` on mobile is correct but wastes space. Consider `grid-cols-2` with smaller text.
6. **Guided tour** — already suppressed below 1024px (good). But inline getting-started cards needed for mobile users.

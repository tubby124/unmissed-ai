# D417 — Universal: physical address field for all niches (agent can't answer "where are you located?")

**Status:** 🔴 Open
**Priority:** HIGH — Universal (affects all niches)
**Date:** 2026-04-05

## Problem
`LOCATION_STRING` in the system prompt is populated from `CITY` which only gets set if the client has a GBP listing. PM clients using zero-data onboarding (no GBP) get:
> "You are Jade, the front desk at Bowness Property Management. You work at a property management company."

No city, no address. If a prospect asks "where's your office located?" — Jade can't answer. This is one of the most common questions callers ask.

NLM finding: "No physical street address field → can't answer 'where are you located?'" — listed as a universal gap across ALL niches.

## Solution

**Onboarding:** Add a physical address / city field to the general info step (step 1 or step 2). Not required, but prompted:
```tsx
<Label>Business address (optional)</Label>
<p>Used when callers ask "where are you located?"</p>
<input placeholder="123 Main St, Calgary AB T2N 1A1" onChange={e => onChange('businessAddress', e.target.value)} />
```

**Settings:** Expose the existing `city` + `state` DB fields as an editable "Business Location" field in AgentConfigCard or AgentOverviewCard.

**Prompt injection:** The intake-transform / provision route should set:
- `CITY` from GBP data OR from manually entered city
- Add `ADDRESS` variable if street address is provided
- Update HOURS/LOCATION block in template-body.ts to use `{{ADDRESS}}` if available

## Files
- `src/app/onboard/` — step 1 or a universal field
- `src/lib/intake-transform.ts` — map businessAddress to CITY + ADDRESS
- `src/lib/prompt-config/template-body.ts` — add {{ADDRESS}} to HOURS/LOCATION section
- `src/lib/prompt-slots.ts` — populate ADDRESS from intake

# D376 — Telegram must be gated/required at onboarding

## Problem
Onboarding completes with telegram_chat_id=null. Agent takes real calls. Nobody is notified. Agent is completely useless.
This is the entire value prop — without Telegram, there's no lead, no notification, no reason to pay.

## Fix
Telegram setup must happen AT or BEFORE activation. Options:
A) Add Telegram setup as a required step in onboarding (between Plan and Launch) — show QR code + instructions. Can skip with acknowledgment "I'll set this up later" but show warning.
B) Activation step blocks with amber warning if telegram_chat_id=null: "Your agent won't notify you without Telegram — connect now or set up from dashboard."

Option B is lower effort and doesn't break the 3-step flow.

## Files
- `src/app/onboard/steps/step6-activate.tsx` — add Telegram warning if not connected
- `src/app/api/provision/trial/route.ts` — maybe send setup instructions via email on provision

## Fix Applied (2026-04-06) — Silent failure resolved

Root cause: `email_notifications_enabled = false` was set at provision time when user picked "Telegram" as notification method, but `telegram_chat_id` isn't connected until after provisioning. Result: zero notifications.

Fixed in both routes:
- `src/app/api/provision/trial/route.ts` — email always enabled at provision (`email_notifications_enabled: null`)
- `src/app/api/stripe/create-public-checkout/route.ts` — same fix

Email now always fires as fallback. Telegram fires when the bot connects asynchronously. No more silent failures.

## Remaining Gap — Hard gate not implemented
Option B (amber warning blocking the Launch button if not connected) is NOT yet built. The current step6-activate.tsx shows inline instructions but doesn't block launch. Deferred — silent failure was the critical risk, not the UX gate.

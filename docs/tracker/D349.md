# D349 — Orb as global loading indicator

## User feedback
"After uploading the doc it should really be the orb that's the loading icon globally everywhere."

## Current state
- The AI Compiler "Analyzing with AI..." screen uses a static blue star/orb icon
- The test call uses the animated WebGL voice-powered orb (FloatingCallOrb)
- Onboarding provision uses a text spinner
- Various loading states use different spinners/skeletons

## Vision
The voice-powered orb should be THE loading/processing indicator across the entire product:
- **Knowledge upload**: Orb pulses while "Analyzing with AI..."
- **Intelligence generation**: Orb on GBP card or Launch step while Haiku works
- **Provisioning**: Orb animates during agent creation
- **Website scrape**: Orb on Knowledge page during extraction
- **Any AI-powered operation**: Orb signals "AI is working"

This creates a consistent brand signal: "when you see the orb moving, AI is thinking."

## Implementation
1. Extract a lightweight `ProcessingOrb` variant from `FloatingCallOrb` (no WebRTC, just animation)
2. Accept `state` prop: 'idle' | 'processing' | 'success' | 'error'
3. Replace all loading spinners in AI-powered operations
4. Keep the full WebGL orb for actual voice calls

## Files
- `src/components/ui/voice-powered-orb.tsx` — existing WebGL orb
- New: `src/components/ui/processing-orb.tsx` — lightweight variant
- Update: FileUploadPanel, AgentIntelligenceCard, provision loading, scrape loading

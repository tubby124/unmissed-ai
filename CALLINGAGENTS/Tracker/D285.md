# D285 — Prompt Sandwich Framework

## Vision
System prompt structured top-to-bottom like an essay sandwich:

```
TOP BREAD — Identity + Safety (non-negotiable, always present)
  - Who you are, who you work for
  - Hard safety rules (no impersonation, no medical advice, hang-up)
  - Personality line (from voice preset)

FILLING — User-Configured Sections (all from template variables)
  - TRIAGE_DEEP (from caller reasons → Haiku → routing script)
  - SERVICES_OFFERED (from service catalog)
  - CALL HANDLING MODE (from agent_mode selection)
  - INFO COLLECTION (what to collect per intent)
  - Conditional sections: BOOKING FLOW, SMS FOLLOW-UP, TRANSFER

BOTTOM BREAD — Edge Cases + Closing (non-negotiable)
  - How to handle unknowns
  - Closing/goodbye behavior
  - GLM-4.6 safety rules (rules 12-14)
```

## Key Principle
The "bread" is us (safety, structure). The "filling" is 100% the user's data injected as variables. User designs the filling without knowing they're designing a prompt.

# D340 — Menu knowledge upload → agent answers menu questions

## Problem
Intelligence seed gives the agent ROUTING (TRIAGE_DEEP) but not KNOWLEDGE (menu, prices, ingredients). When a caller asks "how much is a large pepperoni?", the agent says "i'm not sure" and routes to callback.

Promptfoo results: 3 knowledge tests FAILED (pizza prices, garlic knots, gluten-free).

## Root cause
Menu data exists in `clients/sals-ny-pizza/domain-knowledge.md` but is NOT:
1. Uploaded to `knowledge_chunks` (pgvector)
2. Injected into `business_facts` or `extra_qa`
3. Available via `queryKnowledge` tool at call time

## Verification steps
1. Start dev server
2. Log into dashboard as admin or test client owner
3. Go to Knowledge page
4. Upload `domain-knowledge.md` text content via FileUploadPanel or KnowledgeTextInput
5. Verify chunks appear in ChunkBrowser with status='approved'
6. Verify `queryKnowledge` tool is registered (check `clients.tools` in Supabase)
7. Make a test call (WebRTC from dashboard) and ask "how much is a large pepperoni?"
8. Agent should answer with correct price ($28 for 20")

## Alternative: business_facts injection
For small menus (<20 items), menu can go into `clients.business_facts` array → injected at call time via `{{businessFacts}}` templateContext. This bypasses pgvector entirely.

Test: Update `business_facts` on the test client with menu items, make a test call, verify agent quotes prices.

## Promptfoo re-validation
After knowledge is uploaded, re-run:
```bash
promptfoo eval -c tests/promptfoo/sals-ny-pizza-test.yaml
```
Target: menu knowledge tests (pizza prices, garlic knots, gluten-free) should PASS.

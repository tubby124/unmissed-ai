# D353 — Dashboard WebRTC test calls don't appear in call logs

## Bug
Browser test calls from the dashboard prompt lab didn't insert to `call_logs`. User tests agent 3x → "No calls yet" → thinks nothing works.

## Root Cause
Only `browser-test-call/route.ts` was missing the insert. The other two routes already had it:
- `agent-test/route.ts` — already inserts with `call_status: 'test'`
- `trial/test-call/route.ts` — already inserts with `call_status: 'trial_test'`

## Fix (2026-04-01)
Added fire-and-forget `call_logs` insert to `browser-test-call/route.ts`:
- `ultravox_call_id` from created call
- `client_id` from `targetClientId`
- `call_status: 'test'`
- `caller_phone: 'webrtc-test'`
- `started_at: new Date().toISOString()`
- Uses `.then()` pattern — does not block response

## Files
- `src/app/api/dashboard/browser-test-call/route.ts` — added insert

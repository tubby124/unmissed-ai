# D398 — GREETING_LINE Fallback Order Fix

**Status:** ✅ Done  
**Priority:** P0  
**Date:** 2026-04-04

## Problem
GREETING_LINE fallback fired before the < 20 char validator. Short Haiku values passed the fallback but were then cleared by the validator, resulting in an empty GREETING_LINE in the final prompt.

## Fix
Reordered: validator now runs first (clears short values to ''), then fallback fires if empty.

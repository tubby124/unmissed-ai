# D368 — Retrieval instruction fails when zero inline facts

## Problem

`buildRetrievalInstruction()` in `src/lib/knowledge-retrieval.ts` produces an instruction that assumes inline `Key Business Facts` exist above it. When `business_facts` is null and `extra_qa` is empty, the instruction says:

> "The Key Business Facts above are authoritative. Use queryKnowledge only for details not already covered."

There are no facts above — so the agent interprets this as "everything is covered" and never calls `queryKnowledge`, even though 15+ approved knowledge chunks exist in pgvector.

## Root Cause

`buildRetrievalInstruction()` has a `knowledgeTruncated` boolean but no `hasInlineFacts` boolean. It always writes the "facts above are authoritative" preamble regardless of whether any facts were injected.

## Fix

In `buildRetrievalInstruction()` (line 118), add a parameter for whether inline facts exist. When zero facts:

```
## KNOWLEDGE LOOKUP
You have access to a knowledge base with detailed business information.
No pre-loaded facts are available — use queryKnowledge for ALL caller questions about services, pricing, hours, policies, or business details.
If queryKnowledge returns a relevant result, answer naturally. If it returns no results, tell the caller you will have someone follow up — never guess.
Do NOT use queryKnowledge for greetings or emergencies.
```

## Files
- `src/lib/knowledge-retrieval.ts` — `buildRetrievalInstruction()` line 118
- `src/lib/__tests__/knowledge-retrieval.test.ts` — add zero-facts test case
- `src/lib/agent-context.ts` — pass fact count to retrieval config

# D281 — Owner Name (CLOSE_PERSON) Editable Post-Onboarding

## Problem
Agent says "I'll let Emon know" but the owner's actual name isn't Emon. There's no UI to change the CLOSE_PERSON / owner name after onboarding. The name is baked into the system prompt at generation time.

## Fix Plan
- Add `owner_name` field to settings UI (Identity card or new card)
- On save: `patchAgentName()`-style replacement for the close_person reference in the prompt
- Or better: make CLOSE_PERSON a template variable `{{CLOSE_PERSON}}` that's resolved at generation and patchable post-generation
- Store canonical owner name in `clients.owner_name` (column may already exist from provision)

## Status
Open — quick win, high user impact.

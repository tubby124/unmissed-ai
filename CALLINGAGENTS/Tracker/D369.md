# D369 — Old-prompt banner on dashboard for pre-Phase-1 clients

## Problem

Clients provisioned before the prompt architecture refactor (Phases 1-6) have old-style prompts with:
- No section markers (`<!-- unmissed:* -->`)
- No `{{templateContext}}` placeholders (added by updateAgent but not native)
- No HOT/WARM/COLD triage rules in the prompt
- No PRIMARY GOAL
- No knowledge retrieval instruction
- No slot-based regeneration support

Dashboard settings edits (services, business name, etc.) that rely on section patchers or slot regeneration silently fail or produce no effect on these clients. The user has no way to know this.

## Fix

Add an "old prompt" detection + warning banner on the Settings page.

**Detection:** check if `system_prompt` contains `<!-- unmissed:` — if not, it's old-style.

**UI:** amber/warning banner at top of Settings page:
> "This agent uses a legacy prompt format. Some settings changes may not take effect. Contact support to migrate to the current system."

Optionally: a "Regenerate Prompt" button (calls `recomposePrompt()` with current DB values) — but this needs D304 migration safety gates first.

## Files
- `src/components/dashboard/settings/AgentTab.tsx` — add banner
- API: add `prompt_format: 'legacy' | 'current'` to settings GET response (or compute client-side from prompt text)
- `src/lib/prompt-patcher.ts` — already has `hasMarkers()` check that can be reused

## Depends On
- D304 (old-client migration) for the "Regenerate" action
- D368 (retrieval instruction fix) for knowledge to work post-migration

## Affected Clients
- `e2e-test-plumbing-co` (test client — safe to experiment on)
- Potentially any client provisioned before 2026-03-15

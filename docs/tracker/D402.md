# D402 — Complete property-management chips (types, maintenance def, excluded)

**Status:** ✅ Done
**Date:** 2026-04-04

## What
Added 3 new chip selectors to `property-management.tsx`:
1. **propertyTypes** (multi-select): Residential, Commercial, Strata/Condo, Mixed — replaces single-select `propertyType`
2. **maintenanceEmergencyTriggers** (multi-select): renames from `emergencyTriggers`, same P1 emergency options
3. **servicesNotOffered** (multi-select): Pest control, Major renovations, Owner disputes, Legal/eviction advice, Commercial properties, Short-term/Airbnb rentals

Updated:
- `prompt-slots.ts`: PM section now handles `niche_propertyTypes` (multi) for INDUSTRY and `niche_servicesNotOffered` → `SERVICES_NOT_OFFERED`
- `intake-transform.ts`: reads `maintenanceEmergencyTriggers` with backward-compat fallback to `emergencyTriggers`

## Why
NLM Q2 (notebook 8d82f254) identified that PM agents sound generic because the onboarding doesn't capture property-specific operational constraints. The agent needs to know what type of properties it manages, what counts as a P1 emergency, and what it explicitly won't handle — so it can route and deflect accurately.

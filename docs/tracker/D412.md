# D412 — PM pricing problem: transfer is Pro-only but PM's headline feature requires it

**Status:** 🔴 Open — Needs Decision
**Priority:** HIGH
**Date:** 2026-04-05

## Problem
`plan-entitlements.ts` gates `transferEnabled` to Pro ($229/mo) only. Core ($119/mo) has `transferEnabled: false`.

The PM niche's headline feature is **24/7 emergency dispatch with live transfer to the emergency tech at 2am**. Without `transferEnabled`, Jade can detect a P1 emergency but CANNOT transfer — she takes a message only.

This means:
- A PM on Core can run Jade as an AI receptionist (message-taking, classification, Telegram alerts)
- But the feature that would make them say "I can't live without this" (live emergency dispatch) requires Pro at $229/mo
- Most small PM companies (10-50 units) will balk at $229/mo before they've proven ROI

Trial has `transferEnabled: true` — so they can EXPERIENCE the full P1 flow, then get cut off when they convert to Core. This creates a cliff.

## Options

**Option A — Unlock transfer for Core on PM niche only**
Add PM niche exception in `plan-entitlements.ts` or in `buildAgentTools()`:
```ts
// PM niche clients on Core get transferEnabled regardless of plan
if (niche === 'property_management' && plan === 'core') {
  return { ...CORE_ENTITLEMENTS, transferEnabled: true }
}
```
Pro: PM at $119/mo gets the full product. Con: This complicates the entitlement model.

**Option B — PM-specific pricing tier**
Create a `pm_core` plan at $149/mo that includes transfer (positioned as "Property Management AI Receptionist"). Higher than Core but lower than Pro. PM companies expect to pay more for specialized software.

**Option C — Move transfer to Core for all niches**
The simplest fix. Upgrade Core to include transfer. The current distinction is "Pro is for high-volume" — but if Core companies need transfer too, reconsider.

## Recommendation
Option A is the quickest fix. Option B is the right long-term move (PM is a separate vertical). Decide before selling to PMs.

# D244 — Intent Coverage UI in Call Analytics

## Problem
No visibility into whether the agent successfully routed by intent. We see call duration and classification (HOT/WARM/COLD/JUNK) but not: "Did the caller want to book but leave without booking?"

## Action
Add intent coverage column to calls table:
- `detected_intent` field in `call_logs` (populated by completed webhook AI summary)
- `intent_fulfilled` boolean (did the agent execute the appropriate flow?)
- Dashboard calls table: Intent badge + Fulfilled indicator

## Files
- `src/app/api/webhook/[slug]/completed/route.ts` — AI classification prompt
- `src/components/dashboard/calls/` — calls table
- DB migration: `call_logs.detected_intent` + `call_logs.intent_fulfilled`

## Acceptance
- Calls table shows "Wanted: Booking | Got: Info" style indicator
- Weekly digest includes intent fulfillment rate

# D296 — Agent Health Score Dashboard

## Problem
No way to know if your agent is performing well. Users toggle settings and hope for the best. No feedback loop on agent quality.

## Vision
Health score (0-100) composed of:
- **Knowledge coverage:** % of caller questions that have answers (from knowledge_query_log gaps vs total queries)
- **Completion rate:** % of calls where all COMPLETION_FIELDS were collected (from call_logs.call_state)
- **Caller satisfaction proxy:** % of calls that lasted > 30s AND collected at least 1 field (vs hang-ups)
- **Response gaps:** count of unanswered questions this week
- **Knowledge freshness:** days since last knowledge update

Displayed as: single score + letter grades per dimension + trend arrow + specific actions to improve.

## Technical Notes
- Most data already exists in call_logs, call_insights, knowledge_query_log
- Need: aggregation queries (could be cron or on-demand)
- Need: HealthScoreCard component on overview page
- D203 (agent performance analytics) is related but broader — this is the user-friendly surface

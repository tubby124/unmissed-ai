# D260 — Service catalog → agent runtime sync

## Problem
`client_services` edits (POST, PATCH, DELETE, apply) don't flow to the live agent. The prompt's FAQ line "What services do you offer?" stays stale after service CRUD.

## Solution (Phase 4)
Created `src/lib/service-catalog-sync.ts`:
- `syncServiceCatalogToPrompt(clientId)` — reads active services, formats via `formatServiceCatalog()`, patches prompt via `patchServicesOffered()`, syncs to Ultravox
- Wired into all 4 service routes as fire-and-forget
- Prompt version tracked via `insertPromptVersion()`
- `clients.services_offered` always updated in DB

## Knowledge Reseed (D300 — Phase 5)
- pgvector knowledge chunks now also reseeded after service CRUD
- `reseedKnowledgeFromSettings()` called when `knowledge_backend='pgvector'`
- Service changes now visible to queryKnowledge tool

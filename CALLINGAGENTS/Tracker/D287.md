# D287 — Niche-Adaptive Onboarding (chips, checkboxes, pre-populated)

## Problem
Current onboarding is one-size-fits-all. A dental office and a plumber answer the same generic questions. The niche-specific fields exist in code (`niche_emergency`, `niche_bookingType`, `niche_newPatients`) but the UX doesn't adapt — it's one long form with text boxes.

Services are collected as free text that gets parsed. Users type into a box. This should be checkboxes/chips pre-populated from the niche.

## Vision
Pick your niche → the form transforms:
- **Plumber picks niche:** sees chips for "Drain Cleaning, Water Heater, Burst Pipe, Bathroom Reno, Sewer Backup" — toggle on/off, add custom
- **Dental picks niche:** sees "Cleaning, Filling, Root Canal, Whitening, Emergency" — toggle, add custom
- **Barbershop:** "Haircut, Beard Trim, Hot Towel Shave, Kids Cut" — toggle, add custom

Same pattern for:
- FAQ pairs: "Here's what callers usually ask about [niche] — edit or confirm"
- Pricing approach: dropdown ("Never quote / Give ranges / Quote from price list")
- Emergency handling: "Do you handle after-hours emergencies?" (yes/no/business hours only)

All pre-populated from NICHE_DEFAULTS + buildNicheFaqDefaults(). Owner confirms or overrides.

## Three Tiers (from Ownership Model)
1. **"Decide for me"** — pick niche, click through, all pre-filled. 90% ready.
2. **"Let me tweak"** — show pre-filled values, edit what's wrong, confirm the rest.
3. **"Here's my stuff"** — upload docs/website, AI extracts, populate same fields. Owner confirms.

## Data Sources for Pre-Population
- `NICHE_DEFAULTS[niche]` — service lists, FAQ, industry label, capabilities
- `buildNicheFaqDefaults(niche)` — 10-13 Q&A pairs per niche
- D247 Haiku inference — TRIAGE_DEEP from business description
- D246 context extractor — prices/policies from website scrape
- AI Compiler — structured extraction from uploaded docs

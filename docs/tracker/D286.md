# D286 — Dashboard + Onboarding UI Alignment

## Problem
The current dashboard UI was built for the old monolithic prompt model. It doesn't match the new ownership model (bread vs filling, user designs the prompt via fields).

### Current gaps
- Settings cards edit DB fields but don't visibly connect to "what your agent knows"
- Onboarding collects data but doesn't show users what their agent will do with it
- No "Agent Brain" or "What Your Agent Knows" surface exists
- The 19 sandwich slots are invisible — users see disconnected settings cards
- The 3-tier onboarding ("decide for me" / "let me tweak" / "here's my stuff") isn't implemented
- Some fields exist on dashboard but don't flow to the prompt or agent (fake controls)

## Solution
Phase 5 bridges the gap:
1. Group variables by slot on the dashboard (Identity, Tone, Goal, Flow, Knowledge, etc.)
2. Label every variable with what it does ("What your agent asks callers")
3. Show "what your agent knows" in a single surface (precursor to Agent Brain in Phase 6)
4. Prepare the UI skeleton for Phase 6's Agent Brain page

# D255 — Guided Context Data Entry at Onboarding

## The Problem
`context_data` is the most powerful injection (prices, policies, urgency keywords) but:
- D246 extracts it from the website automatically via Haiku
- If there's no website, or the website is sparse, `context_data` is empty
- Empty `context_data` = agent can't answer "how much?" or escalate emergencies
- Currently no fallback prompt asking the owner to fill it in manually

## What This Is
After the website scrape step (or if no website URL provided), show a simple form:
1. "What do your services typically cost?" (free text, e.g. "Windshield replacement: $300-450, Chip repair: $80-120")
2. "Any urgent situations callers might describe?" (e.g. "flooding, no heat, ADAS recalibration required")
3. "Any policies callers ask about?" (e.g. "We accept ICBC, we do mobile service anywhere in Metro Vancouver")

This gets formatted into the same `PRICES\n...\n\nPOLICIES\n...\n\nURGENCY WORDS: ...` structure as D246.

When no website scrape: always show this. When scrape ran: show pre-filled with extracted content + let owner correct/add.

## Files
- `src/app/onboard/steps/step3-capabilities.tsx` or a new step
- `src/lib/intake-transform.ts` — already wires `contextData` from `websiteScrapeResult.contextData`
- Add fallback: if `websiteScrapeResult.contextData` is null → use manually entered values

## Why
Windshield Hub works because Sabbir's pricing is in context_data. Without it, the agent says "Sabbir will call you back with a price" instead of "chip repairs start around $80-120, want to book?" The latter closes the job. The former loses the customer to whoever answers the phone first.

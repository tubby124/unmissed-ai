# D414 — PM trial aha moment: pre-scripted test scenario

**Status:** 🔴 Open
**Priority:** MEDIUM
**Date:** 2026-04-05

## Problem
Trial users land on the dashboard after onboarding and see "Make a test call" with a generic placeholder. A PM doesn't know what to say. They might test a random query, not trigger P1 triage, and leave thinking "this is just a voicemail."

The aha moment for PM is: saying "my pipe just burst" and hearing Jade immediately shift to urgent mode, collect name + unit, and attempt transfer (or confirm it would transfer). That takes 20 seconds and proves the product's value.

## Solution
In the test call UI (AgentTestCard or TestCallCard), add PM-specific scenario prompts for niche === 'property_management':

```
Try these scenarios with Jade:
1. 🚨 "Hi, my pipe just burst" → P1 emergency flow + transfer attempt
2. 🔧 "The furnace stopped working" → P2 urgent routing
3. 🏠 "I saw your 2-bedroom listing" → Showing request collection
```

On click of a scenario → auto-pre-fill a suggested opening line or show it as a hint label, so the user knows what to say.

Bonus: After each test call, show a "What just happened" panel that maps the call to the Jade feature it demonstrated. "Jade detected a P1 emergency and would have transferred to your emergency tech number."

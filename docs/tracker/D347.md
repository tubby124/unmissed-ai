# D347 — Fix CLOSE_PERSON fallback

## Problem
`provision/trial/route.ts` line 91 had: `(data.ownerName || data.businessName || '').split(' ')[0]`
When GBP doesn't provide an owner name, this extracted "Red" from "Red Swan Pizza" as CLOSE_PERSON. Agent says "I'll have Red call ya back" — meaningless.

## Fix
Changed to `data.ownerName?.split(' ')[0]?.trim()` — only uses actual owner name. If absent, CLOSE_PERSON is omitted and the prompt template handles it gracefully (falls back to "the team").

## Status: DONE ✅

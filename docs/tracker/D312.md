# D312 — Settings Page Bento Grid Layout

## Problem

Settings page used `max-w-3xl mx-auto` (768px max) while the Overview page had no max-width constraint — full width. This made Settings feel narrow and cramped.

Additionally, AgentTab used `space-y-2` (8px gaps) with 6 collapsible `SettingsSection` accordion wrappers. Cards were hidden behind accordions and stacked in a single column. This was the opposite of the Overview's spacious multi-column bento layout.

## Fix

### Container Width
- `SettingsView.tsx:180` — Removed `max-w-3xl mx-auto` entirely. Settings now uses the same full-width layout as Overview. Changed to `px-4 sm:px-6 py-6 space-y-5`.
- Non-admin top section — Changed from `grid-cols-1 sm:grid-cols-2 gap-3` to `grid-cols-1 md:grid-cols-2 xl:grid-cols-3 gap-4`. Capabilities card gets `md:col-span-2 xl:col-span-2`.

### Bento Grid (AgentTab)
- Replaced `<div className="space-y-2">` with `<div className="grid grid-cols-1 md:grid-cols-2 xl:grid-cols-3 gap-4">`
- **Removed all 6 `<SettingsSection>` accordion wrappers** — cards are now always visible
- Added section label dividers (`col-span-full pt-2` with uppercase text): Identity & Voice, Knowledge, Capabilities & Routing, Agent Script, Configuration, Plan & Billing
- Wide cards get `md:col-span-2` or `col-span-full`: CapabilitiesCard, AgentOverviewCard, CallRoutingCard, PromptEditorCard, AdvancedContextCard, ActivityLog
- SettingsPanel drawer (Hours) kept outside the grid — still functional

### Layout Structure
```
[Capabilities ─────── span-2] [Test Call]
── IDENTITY & VOICE ──────────────────────
[AgentOverview span-2 (admin)] [CallMode]
[AgentMode] [VoiceStyle] [Voicemail]
── KNOWLEDGE ─────────────────────────────
[Services] [AgentKnowledge] [AdvContext span-full (admin)]
── CAPABILITIES & ROUTING ────────────────

[PlanInfo] [Billing] [SetupProgress] [Phone]
```

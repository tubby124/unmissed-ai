# D351 — Overview Knowledge card stale after knowledge upload

## Bug
The Overview page Knowledge Base card shows:
- "WHAT YOUR AGENT KNOWS: 10 facts · 4 Q&A" — this is from `clients.business_facts` and `clients.extra_qa` (GBP data seeded at provision)
- Source list: Website=1, AI Compiler=1, Documents=empty
- Does NOT reflect the 12 menu items just uploaded via AI Compiler

Meanwhile the Knowledge page correctly shows:
- "27 chunks live · 100% coverage"
- Website: 15 chunks active
- AI Compiler: 12 chunks active

## Root cause (likely)
The Overview card reads from `clients.business_facts` / `clients.extra_qa` (the settings-based knowledge) but NOT from `knowledge_chunks` table (the pgvector knowledge). These are two separate knowledge paths:
1. `business_facts` / `extra_qa` → shown in "What your agent knows" on Overview
2. `knowledge_chunks` → used by `queryKnowledge` tool at call time, shown on Knowledge page

The Overview card should show BOTH — or at least show total chunk count and a "12 menu items from AI Compiler" line.

## Additional issues
- AI Compiler shows "1" on Overview (meaning 1 document/batch) but "12 chunks active" on Knowledge page — confusing count mismatch
- Documents section on Overview says "empty" / "No approved chunks yet" even though the upload went through AI Compiler, not Documents pipeline

## Fix
1. Overview Knowledge card should query `knowledge_chunks` count by source and show it
2. AI Compiler count should show chunk count (12) not batch count (1)
3. Or: unify the display to show "27 knowledge items" total with source breakdown

## Files
- Overview page component — Knowledge Base card section
- Knowledge page component — source count queries (working correctly, reference implementation)

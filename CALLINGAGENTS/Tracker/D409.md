# D409 — Progressive Disclosure in PM Onboarding

## Problem
PM onboarding screen shows all fields at once (tenant roster, emergency contacts, pet policy, etc.). Low-value fields clutter the screen and increase drop-off for new PM signups.

## Fix
Show only critical fields first (property name, emergency phone, tenant roster). Reveal secondary fields (pet policy, parking, packages) behind a "More settings" toggle or as a step 2. Fields should auto-save as the user types — don't require explicit submit for each section.

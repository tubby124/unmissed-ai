# D423 — Agent status indicator: is my agent actually online and synced?

**Status:** 🔴 Open
**Priority:** HIGH
**Date:** 2026-04-05

## Problem
A client who makes settings changes has no way to know:
1. Did the change reach the live agent?
2. Is the Ultravox agent actually healthy and answering calls?
3. Was the last sync successful?

Current UX: save settings → success toast → nothing. The client doesn't know if their agent was updated or if it's even online.

This is a trust problem. A PM who updates their emergency phone and calls in an hour later to test it — and finds the old routing — will churn.

## Solution
Add an **Agent Status** indicator, visible on the settings page and/or the dashboard home:

**States:**
- ✅ `Agent active — last synced 2 min ago` — green dot, all good
- ⚠️ `Syncing...` — amber dot, change was saved, Ultravox update in progress
- 🔄 `Changes pending` — blue dot, DB updated but agent sync hasn't run yet
- ❌ `Agent offline` — red dot, Ultravox agent unreachable, calls may not be answered

**Save feedback upgrade:**
Currently: save → "Saved" toast.
New: save → "Saved. Updating your agent..." → "✅ Your agent is updated and ready."

This requires:
1. Track `ultravox_last_synced_at` on the client row (or derive from `updated_at` vs Ultravox state)
2. Surface sync status in the dashboard

## Files
- `src/components/dashboard/settings/AgentOverviewCard.tsx` — add status indicator
- `src/app/api/dashboard/settings/route.ts` — return sync status after PATCH
- `src/lib/database.types.ts` — add `ultravox_last_synced_at` if not present

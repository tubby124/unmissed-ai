# D434 — Generic / thin niche prompt has no triage depth

**Status:** open
**Priority:** P1
**Filed:** 2026-04-06

## Problem

Users who select the 'other' niche — or any niche without a full prompt-slots implementation — get a generic prompt with no triage depth. The agent only collects name, callback number, and reason. It cannot prioritize calls, detect urgencies, or ask intelligent follow-up questions. This makes the product feel like a voicemail box, not an AI receptionist.

## Niches affected

- `other` — no requirements, no defaults, no triage logic
- `voicemail` — by design, but even triage-mode users on this niche get nothing industry-specific
- `print_shop`, `mechanic_shop`, `pest_control`, `locksmith`, `electrician` — have minimal niche-defaults, no TRIAGE_DEEP in prompt-slots.ts

## Root Cause

`niche-defaults.ts` has a `NICHE_DEFAULTS` map. If the niche key is missing or has no `triageDeepRules`, the `buildConversationFlow()` slot falls back to generic INFO_TO_COLLECT. No safety checks, no urgency keywords, no industry-specific questions.

## Fix Options

**Option A (fast):** Build a universal triage template for 'other' — generic questions (what's the issue, how urgent, best callback time). 6 lines in niche-defaults.ts.

**Option B (right):** Require users to enter 3 caller types / triage questions during onboarding (Step 2 caller reasons). These are already collected as `callerReasons` and fed to Haiku to generate TRIAGE_DEEP. Ensure Step 2 validates that at least 1 caller reason is filled for the 'other' niche before advancing.

Option B is already partially built — the caller reasons input exists and Haiku runs on blur. The gap is no canAdvance gate enforcing it for 'other' niche.

## Files

- `src/lib/prompt-config/niche-defaults.ts` — add generic fallback TRIAGE_DEEP
- `src/app/onboard/config/steps.ts` — add canAdvance gate for 'other' niche requiring ≥1 callerReason
- `src/lib/prompt-slots.ts` — confirm triageDeepRules fallback is safe if null

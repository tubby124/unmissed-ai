# D360 — Raise prompt char limit to 25K

## Problem
The intelligence seed generates prompts ~18K chars. The provision route error message referenced a stale 12K limit. The actual `validatePrompt()` already treats size as a warning (not hard reject) with a 25K threshold via `PROMPT_CHAR_HARD_MAX = 25000`.

## Fix (2026-04-01)
1. `provision/trial/route.ts` — updated 422 error message from `hard_limit: 12000` to `hard_limit: 25000`
2. `inbound/route.ts` — updated stale `(max 12000)` log message to `(max 25000)`
3. `knowledge-summary.ts` — updated JSDoc from "hard max 12K" to "hard max 25K"

**Finding:** `validatePrompt()` does NOT hard-reject on size — it's a warning only. If 18K prompts were rejected at provision, the cause is structural validation (missing hangUp, CALLER ENDS CALL, etc.), not character count.

**Remaining:** Several UI components still display "12K" as the limit (PromptEditorCard, RuntimeCard, PromptPreviewCard, PromptEditorModal). Separate polish item.

# D278 — "Agent Brain" Dashboard — Centralized Knowledge Visibility

## Problem
Users can't see EVERYTHING their agent knows in one place. Knowledge is scattered: system prompt has some, context_data has some, knowledge_chunks has some, extra_qa has some, business_facts has some. No single view shows "this is what your agent will say about X."

## Vision
One dashboard section: "What Your Agent Knows"
- Services & pricing (from service catalog + context_data)
- FAQs (from extra_qa + knowledge_chunks)
- Business info (hours, policies, emergency words from context_data)
- Call routing rules (from TRIAGE_DEEP)
- Edge cases & special instructions

All editable inline. Every edit flows through to the live prompt via template variables.

## Dependencies
- D280 (UI-driven prompt composition)
- D283 (all variables visible + editable)

## Status
Open — architectural item. Part of the "user designs the prompt" vision.

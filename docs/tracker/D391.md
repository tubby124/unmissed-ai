# D391 — Re-trigger Haiku Intelligence After StepNiche

## Problem
`generate-agent-intelligence` fired in step 1 (step1-gbp.tsx) BEFORE the user reached the niche step. At that point `nicheAnswers.services` was always empty. Result: Haiku generated TRIAGE_DEEP without knowing what services the business actually offers.

## Root Cause
`fireIntelligenceGeneration` in step1-gbp.tsx used `currentData.selectedServices` (always []) instead of `nicheAnswers.services` (filled in step 2). The two steps were in the wrong order for data to flow.

## Fix
Added `useEffect` in `step-niche.tsx` that:
1. Watches `nicheAnswers.services` for changes (serialized to stable string)
2. Debounces 800ms after last chip toggle
3. Fires `generate-agent-intelligence` with full context: businessName, niche, services (chips), callerReasons, GBP description, websiteScrapeResult
4. Writes result to `agentIntelligenceSeed` + `nicheCustomVariables`

## Result
Haiku now gets the complete picture. TRIAGE_DEEP, GREETING_LINE, and URGENCY_KEYWORDS are generated with actual services context, not generic niche defaults.

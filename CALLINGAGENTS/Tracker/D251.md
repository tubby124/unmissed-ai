# D251 — Per-Section Prompt Editor UI (Triage for Non-Admin)

**Status:** ✅ DONE (2026-03-31)

## What Was Built

Added `SectionEditorCard` for `sectionId="triage"` to non-admin owners in `AgentTab.tsx` under the "What It Can Do" section, after `CallRoutingCard`.

## Why this works without backend changes
- `isSectionEditAllowed('triage', 'owner')` already returns `true` in `lib/settings-schema.ts`
- `replacePromptSection()` in `lib/prompt-sections.ts` already handles `triage` via `SECTION_HEADER_ALIASES`
- `PATCH /api/dashboard/settings` already accepts `section_id + section_content`
- `sectionContent[client.id].triage` is populated from `parsePromptSections(system_prompt)` in `SettingsView.tsx`

## UI placement
Under "What It Can Do" → after `CallRoutingCard` (D254) → before booking section

## Label
- Label: "Call Routing Script"
- Desc: "How your agent identifies why someone is calling and what to do next"
- 8 rows textarea

## Self-serve loop
1. D247 onboarding generates TRIAGE_DEEP via Haiku
2. D254 dashboard lets owner update their 3 reasons → regenerate TRIAGE_DEEP
3. D257 suggest-improvements Haiku identifies specific gaps
4. **D251 (this)** lets owner open the triage section and directly edit the routing script
5. Owner confirms → `replacePromptSection()` deploys live

# D268 — Minimal base prompt + dynamic sections

## Summary
Shrink the system prompt from 12-18K chars to ~4-5K char base (safety + identity + voice + flow skeleton). Everything else becomes dynamic based on client config. Missing data = missing section, not placeholder text.

## Why
Current prompts are monolithic — every client gets the same 18K blob regardless of which features they use. This causes:
- GLM-4.6 lag from oversized prompts (D277 root cause)
- Hardcoded business logic overriding user data (D272)
- Duplicate knowledge in prompt + pgvector (D265)
- Prompt exceeding 12K char hard limit for some niches

## Implementation
- Switch `buildPromptFromIntake()` to use slot composition from D274 instead of monolithic `INBOUND_TEMPLATE_BODY`
- Each slot function returns its section or empty string
- `composePrompt(slots)` concatenates non-empty slots
- Archive `INBOUND_TEMPLATE_BODY` as dead export for 1-week rollback window
- Update golden test snapshots to reflect smaller output

## Dependencies
- **Requires:** D274 (named slots), D265 (KB removal), D272 (constraint removal)
- **Enables:** D283 (variable visibility), D280 (UI-driven composition)
- **Resolves:** D277 (plumber lag — prompt drops below 8K)

## Key Files
- `src/lib/prompt-builder.ts` — rewrite core to use slots
- `src/lib/prompt-config/template-body.ts` — archive/delete
- `src/lib/__tests__/prompt-builder-golden.test.ts` — update snapshots

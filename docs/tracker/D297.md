# D297 — Agent Learning Loop UX (agent gets smarter from calls)

## Problem
D252 detects knowledge gaps. D257 generates AI suggestions. D284 designed the self-improving loop. But there's no unified UX where the user sees "your agent learned X this week" and approves/edits/dismisses.

## Vision
"Agent Learning" feed on dashboard:

```
🧠 Alex learned 3 things this week:

1. "Do you install water softeners?" — asked 4 times, no answer
   → Suggested: "we don't do water softeners but I can refer you"
   [Approve] [Edit] [Dismiss]

2. Callers often ask about Warman service area  
   → Suggested fact: "We service Warman, Martensville, and Saskatoon"
   [Approve] [Edit] [Dismiss]
```

Approved items → knowledge_chunks (status='approved') → agent uses them on next call.
Weekly Telegram digest of what the agent learned.

## Technical Notes
- D252 (gap detection) — DONE, feeds knowledge_query_log
- D257 (AI suggestions) — DONE, generates suggested answers
- D270 (frequent KB query → auto-suggest) — designed
- Need: unified "learning feed" component
- Need: one-click approve → embedChunks() → knowledge_chunks
- Need: weekly digest cron → Telegram summary of learnings

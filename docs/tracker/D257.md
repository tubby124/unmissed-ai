# D257 — Self-Improving Agent Flywheel (suggest-improvements endpoint)

**Status:** ✅ DONE (2026-03-31)

## What Was Built

`POST /api/dashboard/suggest-improvements/route.ts`

Gathers all owner context → calls Haiku via OpenRouter → generates 2-3 specific, confirmable prompt improvements → inserts into `prompt_improvement_suggestions` table → surfaces in `PromptSuggestionsCard.tsx`.

## Context gathered per request
- `business_name`, `niche`, `niche_custom_variables` (TRIAGE_DEEP), `context_data`, `extra_qa`, `call_handling_mode`
- Top 5 unanswered questions from `knowledge_query_log` (last 30 days, `result_count=0`, `resolved_at IS NULL`)
- Frequency-aggregated gap map (normalized lowercase)

## Haiku prompt structure
- Inputs: business context block + unanswered questions with counts
- Output: JSON array `[{ section_id, trigger_type, suggestion_text, evidence_count }]`
- Model: `anthropic/claude-haiku-4-5` via OpenRouter
- `max_tokens: 800`, `temperature: 0.2`, `AbortSignal.timeout(10000)`

## Rate limiting
- 5 generation runs / client / 24h (checked against `prompt_improvement_suggestions.created_at`)

## Downstream surfaces
- `PromptSuggestionsCard.tsx` already polls `GET /api/dashboard/prompt-suggestions` every 60s
- Suggestions appear with "Apply in Prompt Editor →" links
- Owners confirm via D251 triage SectionEditorCard

## Key files
- `src/app/api/dashboard/suggest-improvements/route.ts` (new)
- `src/app/api/dashboard/prompt-suggestions/route.ts` (existing GET/dismiss handler)
- `src/components/dashboard/settings/PromptSuggestionsCard.tsx` (existing UI)

# D378 — Linguistic Identity Anchors per Niche

## Problem
New clients onboard and agents sound generic. Root cause confirmed via persona architecture research: no linguistic identity anchors per niche. Generic agents collect name/phone. Niche-specific agents use technical vocabulary + diagnostic branching.

Research finding (Q2): "Linguistic identity anchors are the single biggest differentiator between a generic agent and a niche-specific one. The vocabulary itself signals expertise before any qualification logic runs."

## Fix (2026-04-04)

### niche-defaults.ts
Added `LINGUISTIC_ANCHORS` string field to 3 production niches only:

| Niche key | Business | Anchors |
|-----------|----------|---------|
| `auto_glass` | Windshield Hub | OEM, ARG, aftermarket, chip, crack, full replacement, ADAS calibration, zero deductible, insurance claim, mobile service, rock chip, windshield, side glass, back glass |
| `property_management` | Urban Vibe | CC&Rs, break clause, Priority 1 maintenance, eviction notice, rent roll, lease renewal, maintenance ticket, vacate notice, security deposit, strata, month-to-month, fixed term |
| `outbound_isa_realtor` | Hasan Sharif | CMA, FSBO, days on market, pre-approval, motivated seller, listing appointment, buyer consultation, MLS, LPMAMA, closing timeline, equity, absorption rate |

All other niches: no `LINGUISTIC_ANCHORS` key → empty string → no injection.

**Note:** The spec referred to the real estate ISA niche as `real_estate_isa` but the actual key in niche-defaults is `outbound_isa_realtor`.

### prompt-slots.ts
1. Added `linguisticAnchors: string` to `SlotContext` interface
2. Wired in `buildSlotContext`: `linguisticAnchors: variables.LINGUISTIC_ANCHORS || ''`
3. Injected into `buildConversationFlow` TRIAGE section before `ctx.triageDeep`:
   ```
   Use these terms naturally when they apply: {anchors}
   ```
   Only injects when non-empty. Token impact: ~35–45 tokens per niche. Under 50-token limit.

## Important
Existing live prompts (Urban Vibe, Windshield Hub, Hasan's agent) are unaffected until Regenerate is triggered — same behavior as D377. Anchors apply to new clients onboarding with these niches going forward.

# D346 — Knowledge upload directly from Overview page

## Problem
After onboarding, the Overview page shows a Knowledge Base card with facts/Q&A counts and source tabs (Website, Facts & Q&A, Text Imports, AI Compiler, Documents). But there's no obvious "Upload menu" or "Add a PDF" button right there. The user has to:
1. Notice the Knowledge tab in the nav
2. Navigate to it
3. Find the upload section
4. Figure out which upload method to use

This is too many steps for the #1 thing a restaurant owner wants to do after seeing their agent: "teach it my menu."

## Observed during live test (2026-04-01)
User (Hasan) said: "I want to be able to upload PDFs directly on the Overview instead of going to Knowledge as it's not easy to understand where to take this."

## Fix
Add a prominent upload CTA to the Knowledge Base card on Overview:
- **"Upload menu / price list"** button — opens a file picker or drag-and-drop zone right on Overview
- Accepts: PDF, TXT, MD, DOCX, images
- On upload: fires the existing knowledge upload pipeline (same as Knowledge page)
- Shows progress inline: "Processing menu..." → "12 facts extracted" → "Added to knowledge"
- After upload: the facts count updates and a success message appears

## Alternative (simpler)
Add a **"+ Add knowledge"** button to the Knowledge Base card that deep-links to the Knowledge page with the upload drawer pre-opened. Less ideal but faster to ship.

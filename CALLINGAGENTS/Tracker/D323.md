# D323 — Radical onboarding simplification (3-step flow)

## Vision
Collapse the 7-step onboarding into 3 steps:

### Step 1 — Find your business (~30 seconds)
- GBP search → auto-fills: name, hours, website, niche, address, phone
- Niche badge visible on confirmed card (D315)
- Male/Female voice toggle (not a grid of 20 voices)
- Agent name auto-set from niche
- Loading orb during GBP fetch (D322)

### Step 2 — Pick your plan (~15 seconds)
- Show the 3 pricing cards (Lite/Core/Pro) — same design as homepage
- Plan determines agent mode (D324):
  - Lite = AI Voicemail (take messages)
  - Core = Smart Receptionist (answer questions + lead capture)
  - Pro = Receptionist + Booking (everything + calendar + transfer)
- Trial banner: "7 days free, all features unlocked"
- Each card shows: price, minutes, key features, what's NOT included
- No separate "agent mode" step needed

### Step 3 — Launch (~30 seconds)
- Email + password
- Phone number for notifications
- Orb animates during provisioning
- "Your agent is live" success screen

### What moves to dashboard (post-onboarding)
- Agent personality/tone → VoiceStyleCard
- Services → Knowledge page
- FAQs → Knowledge page
- Caller reasons / urgency words → Settings or Knowledge
- Schedule / hours → Settings Hours card
- Call forwarding → Settings Transfer card
- IVR → Settings IVR card
- Knowledge upload / website scrape → Knowledge page

### What dies
- Step 2 (Voice preview) — fake control cards, collapses to Male/Female toggle in step 1
- Step 3 (Agent mode) — plan determines mode
- Step 4 (Schedule) — GBP hours auto-fill, edit in dashboard
- Step 5 (Train your agent) — dashboard Knowledge page
- All windshield placeholder problems — no more niche-specific questions in onboarding
- Step 3/5 FAQ duplication — no FAQ in onboarding at all

## Subsumes
D315 (niche badge), D316 (fake controls), D317 (placeholders), D318 (step 3 bloat), D319 (voice simplify), D321 (FAQ dedup), D322 (loading orb) — all addressed by this restructure.

## Dependencies
- D324 (Plan = Mode) — must be decided first
- `/ui-ux-pro-max` — visual redesign of the 3-step flow

## Size
Large — fundamental onboarding restructure. But the result is dramatically simpler code (delete 4 step components, simplify 2 others).

# unmissed.ai Dashboard — Full UX Audit & Vision Document

**Date:** March 24, 2026
**Purpose:** Comprehensive audit of the current dashboard UI/UX, with identified problems and a vision for redesign. Written to be shared with a design-focused LLM (ChatGPT) for actionable UX guidance.

---

## 1. What unmissed.ai Is

unmissed.ai is a B2B SaaS platform that gives small businesses (auto glass shops, real estate agents, salons, etc.) an AI-powered phone receptionist. The AI agent answers calls 24/7, handles FAQs, books appointments, classifies leads, sends follow-up texts, and learns from every conversation.

**Three product tiers:**
- **Voicemail Agent** — Just takes messages and notifies the owner
- **Receptionist Agent** — Answers FAQs, provides business info, classifies leads
- **Full Agent** — Everything above + books appointments, transfers calls, sends SMS, learns over time

**Tech stack:** Next.js dashboard, Ultravox voice AI, Twilio telephony, Supabase database, Railway hosting.

**Key competitor:** AskBenny.ca — simpler interface, fewer features, but much easier to understand at first glance.

## 3. Current Navigation (Sidebar)

**10 items — too many for an SMB tool:**

```
1. Overview
2. Agent (expandable → Overview, Knowledge, Actions)
3. Calls
4. Notifications
5. Go Live
6. Live
7. Leads
8. Advisor
9. Settings (expandable → Billing)
10. Back to Site
--- footer ---
Theme | Take tour | Sign out
```

**Problems:**
- "Go Live" vs "Live" — users don't know the difference
- "Advisor" is a chat assistant — should be a floating widget, not a nav item
- "Back to Site" wastes prime nav space
- "Actions" under Agent duplicates capabilities already accessible via Agent page panels
- "Leads" is thin content — could be a tab within Calls
- AskBenny has ~5 nav items. We have 10+.

**Proposed navigation (5 items):**
```
1. Home          — orb + today's update + stats + recent calls + feature toggles
2. Agent         — identity + voice + capabilities (drill-down panels)
3. Calls & Leads — merged call log + lead pipeline
4. Train         — knowledge + learning loop + "add knowledge" (THE differentiator)
5. Settings      — billing + notifications + account + advanced
--- footer ---
Theme | Sign out
(Tour → first-login only, Advisor → floating chat widget)
```

## 4. Page-by-Page Audit

### 4.1 Overview Page

**What it looks like now:**
- Hero: glowing orb animation (idle state) + agent name "Mark" + subtext + "Start Test Call" button
- Status card: name, Live badge, 54 calls this month, 38/500 minutes usage bar
- 3 stat cards: Hot Leads (1), Bookings (0), Avg Quality (44.9)
- Recent Calls: 5 most recent with classification badges and truncated summaries

**What works:**
- The orb is visually striking and unique — this is a differentiator
- Stats at a glance (calls, minutes, leads) are immediately useful
- Recent calls with classification badges (HOT/WARM/COLD) are scannable

**Problems:**
1. **"Start Test Call" is generic** — doesn't convey the magic. The orb is the product's hero feature. Users should feel like they're about to talk to their agent, not "start a test."
2. **No "Today's Update" anywhere** — the owner's most-requested feature. Quick text override like "Happy Easter Monday, we're closed today" or "Owner is in the mountains but I can help." The agent immediately picks this up.
3. **No feature toggles** — can't quickly enable/disable booking, SMS, transfer from the home page. Have to navigate into Agent → deep scroll.
4. **Stats are passive** — "Hot Leads: 1" is just a number. Should be clickable, leading to the actual lead.
5. **No "what your agent knows" summary** — user wants to see at a glance: "Mark knows 59 things, answered 48 calls, couldn't answer 5 questions this week."
6. **Quality score (44.9) is alarming but offers no guidance** — what does this mean? How do I improve it?

**Vision:**
- Orb stays hero but CTA becomes **"Talk to Mark"** (personalized with agent name)
- Right next to orb or immediately below: **Today's Update** input — textarea + "Enrich with AI" button that rewrites the text to match the agent's voice style
- Below: **Active Features strip** — visual toggles for Booking, SMS, Transfer, IVR. Each clickable to configure.
- Below: Stats row (same 3 cards but clickable — Hot Leads clicks into Leads, etc.)
- Below: Recent Calls (keep as-is, it works well)
- Quality score should show a trend arrow + "Improve: fill 3 knowledge gaps" nudge

### 4.2 Agent Page

**What it looks like now:**
This is the LONGEST page in the dashboard. Full-page scroll includes:
- Mini orb + "Talk to your agent" button (redundant with Overview)
- **Identity & Status:** name, phone number, tagline, filler phrases, greeting toggle, enabled services badges
- **Agent Capabilities card:** collapsible sections for each feature (Take messages, Search knowledge, Business hours, Book appointments, SMS follow-up, Voicemail, Transfer) — each with a > arrow to open a SettingsPanel drawer
- **Agent Context:** hours display, knowledge summary, confidence data
- **Voicemail section:** recording, fallback message
- **Voice Style cards:** 4 personality options (Casual & Friendly, Professional & Warm, Direct & Efficient, Empathetic & Patient) — select one
- **Voice preview:** current voice name (Blake - Cartesia) + "Browse Voice Library" button
- **Capabilities checklist:** what's enabled/disabled with configure arrows

**What works:**
- Capabilities card with > arrows to drill-down panels is the right pattern (just built this)
- Voice style cards are visually appealing
- Enabled services badges give quick status

**Problems:**
1. **Page is ENORMOUS** — easily 3000+ pixels of scroll. Users lose context.
2. **Identity section tries to be summary AND editor** — editable fields mixed with display-only info. Confusing.
3. **Voice style cards are disconnected from audio** — picking "Casual & Friendly" doesn't play a sample. No instant feedback.
4. **System prompt / generated prompt may be visible** — users should NEVER see the system prompt. They don't need to know it exists.
5. **Technical language everywhere:**
   - "Filler phrases" → should be "What Mark says while thinking"
   - "Caller greeting" → "How Mark answers the phone"
   - "Transfer conditions" → "When to connect to a real person"
   - "IVR" → "Phone menu"
   - "Confidence threshold" → meaningless to SMB owner
6. **Voicemail section is buried** below voice style, knowledge, hours — many users won't scroll that far
7. **"Enabled Services" badges are read-only** — clicking them does nothing. Should toggle or open config.
8. **Duplicate test-call CTAs** — orb on Overview AND mini orb here. One is enough.

**Vision:**
- **Split into focused cards** (not one mega-scroll):
  - **Identity card:** Name, phone, avatar/photo — clean, 2-3 fields max
  - **Voice card:** Style picker with AUDIO PREVIEW of each style. Click to hear the difference.
  - **Capabilities card:** Keep the > arrows pattern. Each opens a SettingsPanel drawer.
  - **Voicemail card:** Move up, make prominent. "This is what callers hear when you're unavailable."
- **Remove system prompt visibility entirely** — everything the user touches is a friendly control (text input, toggle, dropdown) that writes to the prompt behind the scenes.
- **"Enrich with AI" button** on every free-text field (tagline, greeting, voicemail message, transfer conditions)

- "Approve" / "Edit & Approve" / "Dismiss" buttons

Each item shows: "Used in X calls" badge to prove value.

**Section 4: Add Knowledge**
- Free-text input: "Type something you want Mark to know"
- Upload a document
- Add a website URL to scrape
- "Enrich with AI" button on all text inputs

**Section 5: Test**
"Ask Mark a question and see how he answers" — existing test tool, relabeled.

## 5. Cross-Cutting UX Principles

These six principles should guide every design decision:

### Principle 1: The System Prompt is Invisible

The system prompt is the engine that powers the AI agent. It's a ~6000 character text blob that defines the agent's personality, knowledge, and behavior. But **users should never see it, edit it, or know it exists.**

Every UI control — name field, voice style picker, hours toggle, FAQ editor, Today's Update — writes to the system prompt behind the scenes via API. Users just see friendly controls with clear labels. The prompt rebuilds automatically.

**Currently violated:** The Agent page may show the generated prompt. "System prompt" language appears in some places. Remove all of this.

### Principle 2: "Enrich with AI" Pattern

Whenever users type free-form text anywhere in the dashboard, offer an **"Enrich with AI"** button that:
1. Takes their rough text ("im in the mountains today cant take calls")
2. Rewrites it in their chosen voice style (casual → "Hey! Our owner's out enjoying the mountains today, but I'm here to help you with anything you need!")
3. Shows before/after so they can approve or edit
4. Writes the polished version into the system prompt

This applies to: Today's Update, FAQ answers, voicemail greeting, business description, transfer conditions, tagline — every free-text field.

**Implementation:** OpenRouter API call with the user's voice style preference as context.

### Principle 3: Progressive Disclosure

Every feature follows this flow:
```
See it → Learn what it does → Enable it → Configure it → Test it → Go live
```

Currently, many features jump from "see a toggle" to "complex settings form" with nothing in between.

**Example of good flow for SMS Follow-up:**
1. See: "SMS Follow-up" in capabilities list with description
2. Learn: "After each call, your agent sends a text to the caller with a summary and next steps"
3. Enable: Toggle on → shows sample text message preview
4. Configure: Edit template, choose which call types trigger SMS
5. Test: "Send a test text to yourself" button
6. Go live: Automatic after configuration

### Principle 4: "Your Agent Gets Smarter Over Time"

This is the competitive moat. AskBenny and others don't have a learning loop. Make it visible:

- **Weekly metric:** "Mark learned 12 new things this week"
- **Gap notifications:** "3 questions came up that Mark couldn't answer — help him learn"
- **Usage proof:** Each knowledge item shows "Used in X calls"
- **Quality trend:** "Call quality improved 15% since you added those FAQs"
- **Weekly intelligence email:** Already exists but needs better in-app tie-in

### Principle 5: 12-Year-Old Language

Every label, description, and tooltip should be understandable by a 12-year-old.

| Current (Technical) | Proposed (Human) |
|---------------------|------------------|
| IVR | Phone menu ("Press 1 for...") |
| Filler phrases | What Mark says while thinking |
| Caller greeting | How Mark answers the phone |
| Transfer conditions | When to connect to a real person |
| Knowledge engine | What Mark knows |
| Confidence threshold | How sure Mark needs to be |
| Agent context | What Mark sees about each caller |
| System prompt | (REMOVE — never shown) |
| QA pairs | Questions & Answers |
| Knowledge chunks | (REMOVE — never shown) |
| Webhook | (REMOVE — never shown) |
| Corpus | (REMOVE — never shown) |
| pgvector / embeddings | (REMOVE — never shown) |

### Principle 6: Preview Before Going Live

Every capability should have a "hear/see it in action" moment before it goes live:

| Feature | Preview |
|---------|---------|
| Voice style change | Play 5-second audio sample of Mark in that style |
| IVR / Phone menu | Play audio: "Press 1 to leave a voicemail, or stay on the line..." |
| SMS follow-up | Show sample text message in a phone mockup |
| Today's Update | "Here's how Mark will say this:" + text preview |
| Booking | Show sample conversation: "I can see Thursday at 2pm is available..." |
| Transfer | Show flow diagram: caller → agent → forwarding number |
| Voice change | Play audio comparison: current voice vs. new voice |

The orb/"Talk to Mark" WebRTC test call is the ULTIMATE preview — the user hears all their changes live. This should be encouraged after every configuration change.

## 9. Summary for Design Guidance

**The one-sentence brief:** unmissed.ai has a powerful AI receptionist engine with learning capabilities that no competitor matches, but the dashboard presents it like a developer tool instead of a product a small business owner would love using.

**The goal:** Make a salon owner, auto glass shop, or real estate agent open this dashboard and think "this is so simple, I love it" — not "what the fuck is a knowledge chunk."

**The constraint:** All the features stay. Nothing gets removed. They just get reorganized, relabeled, and presented with progressive disclosure so users discover them naturally instead of being overwhelmed by them all at once.

**The north star interaction:** User opens dashboard → sees their agent's status + key stats → types a quick update ("closed for Easter") → hits "Enrich with AI" → sees polished version → taps "Talk to Mark" → hears the agent say it perfectly → done in 30 seconds.

## 10. Knowledge Page — Deep Dive (Most Critical Redesign)

This section exists because the Knowledge page generated the most user confusion of any page. Additional detail beyond Section 4.3.

### The Learning Loop is the Premium Feature

The system automatically detects questions callers ask that the agent couldn't answer. These show up as "gaps" (currently labeled "unanswered questions"). Example: "Do you sell wiper blades?" was asked 7 different times. The system surfaces this, suggests an answer, and lets the owner approve it. **No competitor has this.** But the UI buries it.

**Current flow:** Gap appears in a tab → user finds it → clicks "Add as FAQ" or "Add to Knowledge" → item disappears → no feedback that anything happened.

**Ideal flow:** Gap surfaces with urgency ("Asked 7 times!") → user sees AI-suggested answer → edits if needed → hits "Enrich with AI" to polish → approves → counter animates "+1" → "Mark now knows 60 things" → celebration micro-interaction.

### FAQ vs. Knowledge — Users Don't Know the Difference

The current UI offers two options when approving a gap: "Add as FAQ" and "Add to Knowledge." Users have no idea which to pick.

**Proposed distinction (in plain language):**
- **Quick Answer (FAQ):** A short, direct answer the agent says immediately. Best for: "Do you do X?" → "Yes, starting at $Y." These are fast, factual, conversational.
- **Detailed Knowledge:** Longer information the agent can search through and summarize. Best for: pricing tables, service descriptions, policies, technical specs. The agent pulls the relevant part when needed.

**UI suggestion:** When approving a gap, show both options with a one-line description and an example. Or better yet: auto-detect based on answer length. Short answer (< 50 words) defaults to FAQ. Long answer defaults to Knowledge.

### Admin View vs. Client View

Some metrics are valuable for the platform admin but meaningless to clients:

| Metric | Admin sees | Client sees |
|--------|-----------|-------------|
| 59 chunks | Yes — useful for debugging | "Mark knows 59 things about your business" |
| 82 gaps | Yes — system health metric | "5 new questions this week" (only actionable ones) |
| pgvector hit rate | Yes — performance metric | Hidden |
| Embedding model | Yes — tech decision | Hidden |
| Confidence threshold | Yes — tuning lever | Hidden |

### Post-Approval Feedback Loop

When a user approves a gap (adds FAQ or knowledge), the UI currently just removes the item from the list. The user gets zero confirmation that the agent actually learned it.

**What should happen:**
1. Item slides out with a success animation
2. "What Mark knows" counter increments visually (+1)
3. Toast notification: "Mark learned the answer to 'Do you sell wiper blades?' — he'll know next time."
4. If they have "Talk to Mark" available, suggest: "Want to test it? Ask Mark about wiper blades."
5. The knowledge item now appears in the "What Mark Knows" list with a "New" badge

### The "Train Your Agent" Reframe

Instead of presenting knowledge as a database to manage, present it as training:

**Current framing:** "Knowledge Engine — 59 chunks, 82 gaps"
**Proposed framing:** "Train Mark — He knows 59 things. Teach him more."

The page title changes from "Knowledge" to "Train" or "Train Mark." The mental model shifts from "I'm managing a database" to "I'm teaching my employee." This is a small copy change with a huge psychological impact.

## 11. What We Missed — The $1M MRR Layer

The audit above covers layout, navigation, and usability. This section covers the BUSINESS MECHANICS that separate a nice dashboard from a revenue machine.

### 11.1 ROI Dashboard — "Mark saved you $2,400 this month"

No user in the dashboard ever sees the ROI of their subscription. They see calls and leads, but never the dollar story. Add a card to Overview:

```
This month: Mark handled 54 calls
Cost per call with Mark: $0.70
Cost per call with a human receptionist: $12.50
You saved: $637 this month
```

This is the #1 churn killer. When a user thinks "should I cancel?", this card answers with math. Receptionist salary benchmarks are public data — use them.

### 11.2 Upgrade Pressure — Feature-Gated Visibility

Users on Starter/Core see no reason to upgrade. They don't know what they're missing.

**Patterns from top SaaS ($1M+ MRR):**
- **Locked feature shimmer:** Show Pro features greyed out with a lock icon + "Upgrade to Pro" — visible but not usable. Calendly, Notion, and Figma all do this.
- **Missed opportunity nudges:** "3 callers tried to book this week, but booking isn't on your plan. Upgrade to never miss a booking." This turns a feature gate into a revenue-loss story.
- **Usage urgency:** When minutes hit 80%, show a yellow bar. At 95%, red bar. "Upgrade to avoid missed calls." Twilio, Vercel, and every usage-based SaaS does this.
- **Annual plan savings:** "Switch to annual, save $194/year" visible in billing. Most SaaS converts 30-40% of monthly users to annual with a visible savings callout.

### 11.3 Celebration Moments — Emotional Hooks

Top SaaS tools create dopamine hits:

| Milestone | What to show |
|-----------|-------------|
| First call answered | Confetti + "Mark just handled his first call!" |
| 10th call | Badge unlock: "Getting started" |
| 50th call | "Mark is a pro — 50 calls and counting" |
| 100th call | Celebration animation + "Mark handled 100 calls. That's X hours of your time saved." |
| First booking | "Mark just booked an appointment for you while you were [working/sleeping]!" |
| First knowledge gap approved | "Mark learned something new! He'll know the answer next time." |
| 7-day streak | "Mark has answered every call for 7 days straight. No missed opportunities." |

These cost almost nothing to build (toast notifications + simple animations) but create massive emotional investment in the product.

### 11.4 Missed Opportunity Tracking — The Revenue Hook

Currently we track calls the agent handled well. We DON'T surface the calls where:
- Caller asked something Mark didn't know → hung up (lost lead)
- Caller wanted to book but booking isn't enabled (lost revenue)
- Caller wanted a human but transfer isn't configured (lost trust)

Each of these should surface on the Overview page:
```
This week: 2 callers wanted to book but couldn't.
Enable booking → never miss an appointment.
[Enable Booking →]
```

This is the strongest possible upgrade + engagement hook. It's not abstract "you should upgrade" — it's "you literally lost $200 because this feature isn't on."

### 11.5 Call Intelligence — Time Heatmap + Trends

Data the owner would love but doesn't have:
- **Time-of-day heatmap:** "Your busiest time is Tuesday 2-4pm" — justifies the product for every owner who's ever been stuck on a job site during peak call hours
- **Week-over-week comparison:** "23% more calls this week vs. last" — shows business growth
- **Repeat caller detection:** "This number has called 3 times this month" — VIP customer alert
- **Call duration trends:** "Average call is 2.5 minutes, down from 3.8 last month" — agent is getting more efficient
- **Top questions asked:** Word cloud or ranked list — helps the owner understand what callers want

### 11.6 Conversion Funnel Visualization

Right now: 3 separate numbers (calls, leads, bookings) on Overview cards.

Should be a visual funnel:

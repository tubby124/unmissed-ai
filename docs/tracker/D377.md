# D377 — PERSONA block sandwich in prompt system

## Problem
LLM identity drift observed at ~8 turns. No primacy anchor existed — the agent's identity was buried at Slot 5 (IDENTITY), after safety preamble and forbidden actions. No recency anchor at all.

## Root Cause
`template-body.ts` was assumed to be the active template but was dead code since D268 (replaced by slot functions in `prompt-slots.ts`). The slot system had 19 slots with no position 0 or position 20.

## Fix (2026-04-04)
Added two new slot functions to `src/lib/prompt-slots.ts`:

- **Slot 0 — `buildPersonaAnchor(ctx)`**: fires before Safety Preamble. States agent name, business, industry, and a one-line "this identity is fixed" precedence rule.
- **Slot 20 — `buildRecencyAnchor(ctx)`**: fires after VIP Protocol (last slot). Restates agent name + business in 2 sentences as a recency anchor.

Both wired into `buildPromptFromSlots()`. Use `ctx.agentName`, `ctx.businessName`, `ctx.industry` — no new variables.

## Files
- `src/lib/prompt-slots.ts` — added `buildPersonaAnchor`, `buildRecencyAnchor`, updated `buildPromptFromSlots`

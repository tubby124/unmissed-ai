# D442 — Overview Page Full Drift Audit + Fix

## Phase 1 findings (2026-04-30, read-only audit)

Full report: `CALLINGAGENTS/00-Inbox/overview-drift-audit-2026-04-30.md` (28KB).
Snapshots saved to `docs/refactor-baseline/snapshots/2026-04-30-pre-d442/{slug}-system-prompt.txt` (5 files) before any analysis.

- **Pipeline split is 4-vs-1, not 1-of-5:** 4 snowflakes (`hasan-sharif`, `exp-realty`, `urban-vibe`, `windshield-hub`) are legacy-monolithic with **0 slot markers**. Only `calgary-property-leasing` (Brian) is slot-pipeline with **17 markers**. D304's premise was correct in number; D442 plan's "Fix 5 = Urban Vibe alone" was wrong on impact.
- **Brian's bug is NOT the same root cause as the snowflakes.** Snowflake symptom: `regenerateSlots` returns `success:false` on legacy prompts → silent no-op. Brian's symptom is the universal Greeting-tile fake-control (registry says `editable:false` but PATCH route doesn't enforce + UI doesn't gate the Edit button). On Brian's client an edit WOULD propagate via slot pipeline; on the snowflakes it silently dies. Different code paths.
- **Universal `clients.tools.hangUp` missing on all 5 clients** — Section 7 risk #7 of the mutation contract. `clients.tools` is supposed to be runtime-authoritative (`toolOverrides:{removeAll:true,add:client.tools}`), yet every client's deployed Ultravox agent has `hangUp` and the DB array does not. Agents hang up correctly in production anyway → spawned D444 to investigate which of three hypotheses is true.
- **New universal fake-control found:** `PROMPT_VARIABLE_REGISTRY[GREETING_LINE].editable=false` is declared but neither the PATCH backend nor the UI Edit button enforces it. Affects all 5 clients regardless of pipeline. Spawned D443.

## Bug bucket (per `core-operating-mode.md`)
- **Propagation bug** — DB write succeeds, Ultravox sync silently fails for legacy prompts
- **Fake-control bug** — UI toggles show "active" but runtime path can't honor (SMS without Twilio number, Transfer/IVR on WebRTC)

## Root cause (verified)
Variable PATCH at [src/app/api/dashboard/variables/route.ts](src/app/api/dashboard/variables/route.ts) DOES sync to Ultravox via `updateAgent()` (lines 300-354). But `regenerateSlots` returns `success: false` silently when the prompt has no section markers (legacy clients). Lines 218-228 return `{ok: true, warning, promptRegenerated: false}` — DB column updates, Ultravox unchanged, owner gets no signal.

For Urban Vibe: `niche_custom_variables.GREETING_LINE` set (probably by AI generation) → Overview shows new value → `regenerateSlots` fails on the legacy prompt → live agent still says the original "Welcome to…" baked at provision time.

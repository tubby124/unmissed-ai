# D305 — Dry-Run Prompt Preview

## Problem
No way to see what a variable edit or recompose would produce before saving. Risk of surprise prompt changes.

## Solution
Two parts:

### Backend (done)
- `dryRun` option added to `regenerateSlot()`, `regenerateSlots()`, and `recomposePrompt()`
- When `dryRun=true`: builds new content, validates, returns `{ preview, currentPrompt }` without saving to DB or syncing to Ultravox
- `POST /api/dashboard/variables/preview` route with two modes:
  - `{ recompose: true }` — full recompose preview
  - `{ variableKey }` — slot-level preview for a specific variable

### Frontend (deferred to Wave 2)
- Diff view UI showing current vs proposed prompt
- Wired to preview API before save confirmation

## Key files
- `src/lib/slot-regenerator.ts` — dryRun option on all 3 functions
- `src/app/api/dashboard/variables/preview/route.ts` — preview API

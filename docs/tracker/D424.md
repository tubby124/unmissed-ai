# D424 — Niche field registry: document end-to-end data flow so adding fields is a 1-file change

**Status:** 🔴 Open
**Priority:** MEDIUM (refactoring/maintainability)
**Date:** 2026-04-05

## Problem
Adding a new PM chip field currently requires touching 4+ separate files:
1. `property-management.tsx` — onboarding UI chip
2. `intake-transform.ts` — map nicheAnswers key → settings/context field
3. `prompt-slots.ts` — inject value into prompt variable
4. `voicemail-prompt.ts` — use variable in PM prompt text
5. (optional) `niche-defaults.ts` — use in FORBIDDEN_EXTRA/TRIAGE_DEEP

If you miss any one of these files, the field is collected but never used (or used in the prompt but not editable post-onboarding). This is how `maintenanceContacts` ended up collected but not injected (D404).

## Solution
Create a `NICHE_FIELD_REGISTRY` that defines each niche field in one place:

```ts
// src/lib/niche-field-registry.ts
export type NicheFieldDef = {
  key: string                    // e.g. 'petPolicy'
  nicheAnswersKey: string        // e.g. 'petPolicy'
  promptSlot: string             // e.g. 'niche_petPolicy'
  settingsVisible: boolean       // show in PmConfigCard?
  settingsLabel: string          // UI label
  defaultValue: string
  mapToContextData?: boolean     // true for tenant roster, maintenance contacts
  contextDataSection?: string    // e.g. 'MAINTENANCE_CONTACTS'
}

export const PM_FIELD_REGISTRY: NicheFieldDef[] = [
  { key: 'petPolicy', promptSlot: 'niche_petPolicy', settingsVisible: true, ... },
  { key: 'parkingPolicy', promptSlot: 'niche_parkingPolicy', settingsVisible: true, ... },
  { key: 'maintenanceContacts', mapToContextData: true, contextDataSection: 'MAINTENANCE_CONTACTS', ... },
  { key: 'tenantRoster', mapToContextData: true, contextDataSection: 'TENANTS', ... },
]
```

Then:
- `intake-transform.ts` reads from the registry to do the mapping
- `PmConfigCard.tsx` reads from the registry to render the edit UI
- `prompt-slots.ts` reads from the registry to know which slots to populate
- Adding a new field = add one entry to the registry

## Files
- `src/lib/niche-field-registry.ts` (new)
- `src/lib/intake-transform.ts` — refactor to use registry
- `src/lib/prompt-slots.ts` — refactor PM slot injection to use registry
- `src/components/dashboard/settings/PmConfigCard.tsx` (D422) — auto-render from registry
